# Jetson 단일보드 실행환경·배포명세

## 1. Artifact 원칙

- 하나의 Git SHA에서 Jetson application image 하나를 만든다.
- 모든 개발 단계는 같은 image에서 launch profile과 device 권한만 다르게 사용한다.
- release는 mutable tag가 아니라 image digest로 고정한다.
- interface, config, calibration, model artifact hash를 manifest에 기록한다.
- stage 승격은 image 재사용 여부와 관계없이 별도의 승인 기록을 남긴다.

## 2. Release manifest

| 범주 | 필수 값 |
|---|---|
| Source | release ID, Git SHA, build time, 담당자 |
| Jetson | board revision, JetPack, L4T, Ubuntu, kernel, Docker, ROS, RMW |
| Image | registry reference, final digest, base image digest, `linux/arm64` |
| Interface | `step_step_interfaces` version·type hash |
| Config | common·Jetson·actuator·stage config SHA-256 |
| Calibration | camera ID, steering center/min/max, motor direction·limit |
| Model | Agent name·version·artifact hash 또는 `none` |
| Teacher | Teacher/evaluator version·dataset lineage 또는 `none` |
| HW | camera ID, I2C bus, `0x40`·`0x60`, motor·servo 식별 |
| Safety | active profile, throttle limit, known limitations |
| Rollback | 이전 release ID·digest·config·model hash |

| `actuator-bench` | 없음 | 없음 | 필요 | 없음 | `BENCH` |
| `drive-test` | 없음 | 선택 | 필요 | 없음 | `DRIVE_TEST` |
| `rule-replay` | 필요 | 없음 | 없음 | 선택 | 기록용 `RULE` |
| `rule-drive` | 없음 | 필요 | 필요 | 선택 | `RULE` |

## 4. Stage config

stage config는 최소한 다음 값을 고정한다.

- `stage_id`, `profile`, `allowed_sources`
- `drive_enabled=false` 기본값
- throttle·steering absolute limit와 slew limit
- command timeout과 camera/path freshness
- Agent artifact hash·confidence threshold·shadow flag
- rule fallback 허용 여부
- bag recording·episode metadata 경로

I2C address, channel, PWM tick, servo pulse는 actuator config에만 둔다. Agent·Rule config가 raw hardware 값을 알지 않게 한다.

## 5. 배포·승격 Gate

1. 차량 전원 OFF 또는 구동 바퀴를 띄운 상태에서 pull·restart한다.
2. manifest의 image digest와 target digest를 대조한다.
3. restart 직후 `drive_enabled=false`를 확인한다.
4. 현재 단계 이전 Gate의 증거가 모두 존재해야 한다.
5. stage profile에 허용된 device만 mount한다.
6. health check 실패 시 이전 image·config·model 세트를 복원한다.
7. rollback 뒤에도 drive enable을 자동 복원하지 않는다.

Teacher가 새 Agent artifact를 생성해도 즉시 Assist에 배포하지 않는다. 새 artifact는 `agent-replay → agent-shadow → agent-assist` Gate를 다시 통과한다.

## 6. 단계별 rollback

- code/config 문제: 이전 release manifest 전체로 복원한다.
- Agent 회귀: image를 유지할 수 있어도 이전 model hash와 threshold를 복원한다.
- calibration 변경: 이전 calibration hash로 복원하고 actuator-bench부터 재시험한다.
- HW 배선 변경: release rollback으로 해결하지 않고 전원 OFF 실물검사부터 다시 시작한다.
