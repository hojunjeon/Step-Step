# 자율주행 AI 단계요구사항

> 소유 파트: HW/CONTROL. 결함 탐지 AI 파트와 구분한다.

## 1. 권한 원칙

- 모델 이름·데모가 아니라 자체 데이터와 폐루프 증거로 권한을 높인다.
- 자율주행 AI는 독립 Safety Supervisor를 우회하지 않는다.
- 평가 MVP 목표는 Replay·Shadow까지이며 Assist는 필수가 아니다.

## 2. 단계와 Gate

| 단계 | 외부 기능 | 필수 입력·산출물 | Exit Criteria |
|---:|---|---|---|
| 1 기본 차량 | 규칙 기반 주행·안전·로깅 | 동기화 bag, 개입·feedback | `HC-M3` 통과 |
| 2 사전학습 비교 | GNM·ViNT·NoMaD 등 Replay | 같은 episode의 trajectory·지연·메모리 | 재현 가능한 비교표 |
| 3 자체 데이터 학습 | RGB+odometry 작은 head | dataset/version, offline metric | 새로운 장소·날짜에서 baseline 개선 |
| 4 실패 강화 | 개입·우회·복구·센서저하 학습 | failure mode별 dataset | 각 mode의 정답·복구 사례 확보 |
| 5 단계 검증 | Replay→Shadow→Assist | disagreement·개입·충돌·복귀 지표 | 권한별 Go/No-Go 기록 |
| 6 Jetson 최적화 | FP32→FP16, 필요 시 INT8 | 정확도·p50/p95/p99·GPU/RAM/전력 | 동시 부하에서도 안전 주기 유지 |

## 3. Runtime 외부 계약

| 영역 | 요구사항 |
|---|---|
| 입력 | image, camera info, odometry, route goal, timestamp·frame·calibration ID |
| 출력 | 후보 trajectory/waypoint, mode/stop 제안, confidence, validity, model ID |
| Health | warm-up, inference latency, dropped input, timeout, GPU/RAM 상태 |
| Failure | stale·invalid·exception 시 출력 무효, 규칙 Planner 유지 |
| 기록 | 입력 episode, output, rule baseline, 선택 여부, intervention 동기화 |

정확한 tensor, preprocessing, model architecture와 학습 코드는 각 단계 상세설계에서 작성한다.

| Shadow | 없음, 실제 운행 중 기록 | 목표 |

## 5. 금지 조건

자체 episode 없음, TF·timestamp 불량, 규칙 baseline 미완성, failure data 부족, independent stop 미검증 중 하나라도 해당하면 Assist로 승격하지 않는다.
