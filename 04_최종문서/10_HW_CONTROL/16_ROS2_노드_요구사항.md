# ROS 2 노드 요구사항

> 이 문서는 외부 책임과 실행 위치만 정한다. callback, class, 내부 알고리즘은 `[상세설계 단계에서 작성]`이다.

## 2. Raspberry Pi 노드

| Node | 책임 | 입력 | 출력 | 목표 주기 |
|---|---|---|---|---:|
| `wheel_encoder_node` | tick·속도·wheel odom 제공 | encoder | ticks, wheel odom | 50~100Hz |
| `imu_node` | ENU IMU·health 제공 | BNO sensor | `/imu/data` | 50~100Hz |
| `estop_node` | NC E-stop 상태 보고 | contact | `/safety/estop` | event+상태 |
| `actuator_controller_node` | 승인 명령의 유일한 actuator 적용 | target cmd | PWM/DIR/steering, feedback | 50~100Hz |
| `safety_watchdog_node` | command·heartbeat·E-stop 감시와 EN 차단 | safety inputs | EN, RPi safety state | 100Hz 이상 목표 |
| `rpi_monitor_node` | RPi·I/O·loop health 제공 | OS/I/O 상태 | heartbeat, diagnostics | 10~20Hz |

## 3. 소유권 요구사항

- actuator hardware API는 `actuator_controller_node` 하나만 소유한다.
- watchdog EN line은 actuator process와 분리해 `safety_watchdog_node`가 소유한다.
- `odom → base_link` TF broadcaster는 하나만 존재한다.
- test script, Nav2, AI node, `/cmd_vel`은 actuator를 직접 소유하지 않는다.
- 모든 노드는 lifecycle·health·shutdown 기대 동작을 상세설계에서 명시한다.
