# 센서·보정·동기화 요구사항

## 1. 공통 요구

모든 센서 데이터는 timestamp, frame ID, calibration revision과 health를 가져야 한다. driver 수신 시각을 sensor capture 시각처럼 사용하지 않으며, 실제 제공 가능한 timestamp 종류를 기록한다.

## 2. 센서별 계약

| 센서      | 필수 보정·metadata                                | 품질 Gate                            |
| ------- | --------------------------------------------- | ---------------------------------- |
| 전방 카메라  | intrinsic, distortion, 해상도, FPS, 노출, mount TF | reprojection·영상 지연 기준 `[결정 필요]`    |
| 측면 카메라  | 위 항목 + 촬영 영역·방향                               | 대표 프레임에 결함 모형이 유효 크기로 포함           |
| LiDAR   | scan frame, 각도·거리 범위, mount TF                | 정지 물체 거리 오차·drop rate 기준 `[결정 필요]` |
| IMU     | 모델, 축, ENU 변환, bias, covariance               | 정지·회전 시험 통과                        |
| Encoder | PPR, gear ratio, 방향, wheel radius             | 실측 거리·속도 오차 기준 `[결정 필요]`           |
| GNSS    | datum, fix type, covariance                   | 사용 가능 환경·fallback 명확화              |

## 3. 시간 요구

- Jetson과 RPi clock offset·drift를 주행 시작 전과 종료 후 기록한다.
- bag에는 `/clock` 사용 여부와 system time source를 metadata로 남긴다.
- 서로 다른 sensor의 허용 timestamp skew는 센서 융합 시험으로 확정한다.
- clock sync가 허용치를 벗어나면 자율주행 enable을 금지하거나 degraded 상태로 전환한다.

## 4. TF 요구

- 모든 sensor frame은 `base_link`까지 끊김 없이 연결돼야 한다.
- 같은 TF edge를 둘 이상의 node가 발행하지 않는다.
- calibration·기구 변경은 revision을 올리고 관련 bag·시험을 다시 수행한다.
- 차체 footprint에는 돌출 sensor·bracket을 포함한다.

## 5. 인계물

실제 calibration 파일, 측정 절차, 허용 오차 계산은 `HC-M2` 상세설계에서 작성한다. 이 Spec의 완료 증적은 calibration ID와 결과 보고서, TF tree, timestamp 분석표다.
