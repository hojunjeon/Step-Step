# OrinCar 센서·Actuator 보정·동기화 요구사항

## 1. Camera

| 항목 | 요구사항 |
|---|---|
| 식별 | 모델·serial·USB/CSI·device path 기록 |
| Intrinsic | 해상도별 calibration file과 hash 기록 |
| 장착 | `base_link → camera_link` 정적 transform 측정 |
| Timestamp | Jetson 수신 시각과 driver stamp 차이 기록 |
| Quality | blur·노출·drop·frame rate를 bag에 기록 |

camera 실물이 확인되기 전에는 임의 모델명·해상도·FOV를 명세하지 않는다.

## 2. Steering

- 매뉴얼 조립 초기 center 100도를 출발값으로 사용하되 실차 center로 확정하지 않는다.
- 바퀴를 띄우고 servo pulse/min/max와 wheel angle·기구 간섭을 함께 기록한다.
- `steering_norm=-1/0/1`을 calibration table로 PWM tick에 변환한다.
- calibration 범위 밖 command는 clamp가 아니라 fault로 거부한다.

## 3. Throttle

- Motor HAT B channel과 motor polarity를 실물 확인한다.
- `throttle_norm`과 PWM tick·회전 방향만 보정한다.
- encoder가 없으므로 m/s 변환·speed PID calibration은 정의하지 않는다.
- 초기 wheels-up limit은 `0.15`이며 전압강하·발열·전류 측정 후에만 변경한다.
- `drive-test`, `rule-drive`, `agent-assist`는 각각 검증된 별도 throttle limit을 config에 기록한다.
- Agent limit은 Rule Drive에서 검증한 limit을 넘을 수 없다.

## 4. 시간·TF

- 모든 ROS header stamp는 Jetson clock을 사용한다.
- 각 episode에 clock source와 시작·종료 시각을 기록한다.
- 허용 TF는 `base_link → camera_link` 정적 transform이다.
- odometry source가 없으므로 `map`, `odom`, `odom → base_link` 동적 TF를 발행하지 않는다.

## 5. 인계물

- camera calibration YAML·hash·sample image
- steering calibration table·center/min/max·사진
- motor channel·polarity·stage별 limit·전압/전류 측정
- static TF와 frame naming record
