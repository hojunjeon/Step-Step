# OrinCar 보유 HW 시스템 요구사항

> 기준: `OrinCar_Manual.pdf` 3~6쪽, 18~28쪽 · 범위: 현재 보유한 OrinCar 구성만 사용

## 1. 현재 HW 기준선

| 범주 | 보유 HW | 매뉴얼 근거 | 설계 상태 |
|---|---|---|---|
| Compute | Jetson Orin Nano Developer Kit | 19, 25쪽 | 단일 컴퓨팅 보드 `[확정]` |
| Vision | 전방 카메라 1대 | 25쪽 전체 회로도 | 모델·USB/CSI 인터페이스 `[실물 확인 필요]` |
| Throttle | Motor Driver HAT, 온보드 PCA9685 `0x40`, TB6612FNG | 22~23쪽 | B channel로 DC motor 제어 |
| Steering | 외부 PCA9685 `0x60` | 21, 23, 26~27쪽 | MG996R 1개 제어 |
| Drive | GA25-370 DC gear motor 1개 | 4, 6, 23, 27쪽 | 후륜 1 motor → gear → 2 wheel |
| Steering actuator | MG996R analog servo 1개 | 4, 6, 23, 27쪽 | 4.8~6V, 180도 |
| Power | 4-cell lithium battery shield | 22~23, 28쪽 | 5V 표기 출력, 최대 3A 설명 |
| Manual power | boat/rocker switch 1개 | 4쪽 | 배선·차단 범위 미명시, E-stop으로 간주 금지 |
| Chassis | OrinCar chassis·wheel·gear·linkage | 3~19쪽 | 조립·중심 정렬 필요 |

현재 기준선에는 LiDAR, IMU, wheel encoder, GNSS, 거리 센서, gamepad, 독립 비상정지 회로, 외부 watchdog, Safety MCU가 없다. 해당 장치를 전제로 한 ROS Node·Topic·성능 주장은 금지한다.

## 2. 구성 원칙

| ID | 요구사항 | 상태 |
|---|---|---|
| `HW-REQ-01` | Jetson 하나가 카메라, 판단, 명령 검증, I2C actuator 제어를 실행한다. | `[확정]` |
| `HW-REQ-02` | `actuator_controller_node` 하나만 I2C bus와 주소 `0x40`, `0x60`을 소유한다. | `[확정]` |
| `HW-REQ-03` | 상위 Node는 I2C address·PWM tick을 알지 않고 정규화된 `VehicleCommand`만 사용한다. | `[확정]` |
| `HW-REQ-04` | Motor HAT B channel은 throttle, 외부 PCA9685 channel은 steering으로만 사용한다. | `[확정]` |
| `HW-REQ-05` | 부팅·명령 만료·정상 종료 시 throttle 0을 먼저 기록하고 steering은 검증된 center로 이동한다. | `[확정]` |
| `HW-REQ-06` | 실제 속도·조향각·이동거리는 센서가 없으므로 측정값으로 주장하지 않는다. | `[확정]` |
| `HW-REQ-07` | actuator Bench를 통과하기 전 지면 주행을 금지하고, 이후 단계도 운영자 감독·통제구역 Gate를 순서대로 적용한다. | `[확정]` |
| `HW-REQ-08` | 카메라 모델, Jetson device path, I2C bus, 전원·전류는 실물 증거로 확정한다. | `[검증 필요]` |

## 3. 전기·I/O 계약

| 연결 | 초기 기준 | 확인 항목 |
|---|---|---|
| Jetson ↔ Motor Driver HAT | Jetson 40-pin I2C, `0x40` | bus 번호, SDA/SCL continuity, logic voltage |
| Jetson ↔ 외부 PCA9685 | 같은 I2C bus, `0x60` | 실제 address scan, address 충돌 |
| Motor HAT ↔ GA25-370 | B channel | 극성, 방향, 무부하 전류, 정지 시 출력 |
| PCA9685 ↔ MG996R | steering channel `[실물 확인 필요]` | channel, center/min/max pulse, 기구 간섭 |
| Battery shield ↔ Motor HAT/PCA9685 | 매뉴얼의 5V 출력 연결 | 실제 전압, peak current, voltage drop, 발열 |
| Camera ↔ Jetson | `[실물 확인 필요]` | 모델, USB/CSI, `/dev` path, frame rate |

GA25-370 표에는 9V/12V 사양이 있으나 회로도는 battery shield의 5V 출력을 사용한다. 전압·기동 전류·stall current·Motor HAT 허용 전류의 호환성을 측정하기 전에는 지면 구동을 금지한다.

## 4. 구현 전 차단 Gate

| ID | 확인값 | 합격 증거 |
|---|---|---|
| `HW-G01` | Jetson 보드·40-pin header 상태 | 전원 OFF continuity, 부팅 후 I2C scan |
| `HW-G02` | 실제 I2C bus·address | `i2cdetect`에서 `0x40`, `0x60`만 기대대로 확인 |
| `HW-G03` | Motor HAT IC·channel·ON/OFF 동작 | 실물 마킹, 무부하 방향 시험 |
| `HW-G04` | GA25-370 전압·정상·stall current | datasheet 또는 전류 제한 bench 측정 |
| `HW-G05` | MG996R center/min/max | 바퀴를 띄운 상태의 pulse·각도·간섭 기록 |
| `HW-G06` | battery shield 출력 | idle/steering/startup 전압·전류·발열 기록 |
| `HW-G07` | 카메라 모델·인터페이스·device path | 실물 라벨, `v4l2-ctl` 또는 CSI 확인 |
| `HW-G08` | process kill 시 Motor HAT 출력 | throttle 유지 여부 기록; 자동 정지 증거가 없으면 모든 지면 단계는 운영자 즉시 차단 조건으로 제한 |

## 5. 현재 허용 범위

- 기본 허용: 전원 OFF 배선 확인, I2C scan, camera preview, Replay, 바퀴를 띄운 servo center·저출력 motor 시험.
- 단계적 허용: `HC-M2` 이후 간단 주행, Rule Drive, Agent Shadow·제한 Assist를 각각 별도 Gate로 검증한다.
- 계속 금지: 무인 운전, 정량 속도 제어, odometry·장애물 회피·공공 보행로 안전 성능 주장.
- `boat/rocker switch`는 NC E-stop과 동등하다는 증거가 없으므로 Safety 장치로 표기하지 않는다.
