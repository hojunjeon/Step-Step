# Camera 기반 규칙 주행 기능명세

## 1. 위치와 목적

규칙 주행은 간단한 open-loop 주행시험 다음 단계이며 Agent의 비교 기준선이다. camera image에서 검출한 tactile path를 normalized steering·throttle 후보로 변환한다.

현재 입력만으로는 실제 속도·odometry·장애물 거리를 알 수 없다. 따라서 규칙 주행의 목표는 빈 통제코스에서 점자 경로를 따라 저속으로 진행하고, 경로가 불확실하면 정지하는 것이다.

## 2. 승격 순서

| 단계 | 실행 | actuator 권한 | Exit Criteria |
|---|---|---:|---|
| `Rule Unit` | 기록된 `TactilePath`로 함수 시험 | 0 | 경계값·lost·NaN 시험 통과 |
| `Rule Replay` | 동일 bag에서 후보 재현 | 0 | 3회 command sequence·값 재현 |
| `Rule Wheels-up` | camera 입력과 실제 steering/throttle 연결 | 조건부 | 방향·limit·lost 정지 확인 |
| `Rule Drive` | 빈 통제코스 저속 주행 | Gate 통과 후 | 지정 구간 반복·경로 상실 정지 |

각 단계는 앞 단계 증거가 있어야 실행한다.

## 3. 상태

| 상태 | command 동작 |
|---|---|
| `IDLE` | enable false, throttle 0 |
| `TRACKABLE` | center·heading error 기반 steering, 제한 throttle |
| `AMBIGUOUS` | throttle 0, fault가 아닌 재검출 대기 |
| `LOST` | throttle 0, steering center, stop reason 기록 |
| `STALE` | 후보 폐기, Safety fault latch |
| `STOPPED` | 새 stage 승인 전 command 생성 금지 |

## 4. Rule Module Interface

입력은 `TactilePath`, stage config, 이전 command다. 출력은 `SOURCE_RULE`인 `VehicleCommand` 하나다.

내부 구현은 다음 복잡성을 숨긴다.

- confidence threshold와 hysteresis
- center offset·heading error 조합
- steering sign·gain·deadband
- curvature에 따른 throttle 제한
- steering·throttle slew limit
- lost path 정지와 재검출 조건

상위 Module은 raw PWM, servo pulse, I2C address를 알지 않는다.

## 5. 요구사항

| ID | 요구사항 | 인수 기준 |
|---|---|---|
| `NAV-01` | image·calibration ID·timestamp가 일치해야 한다. | 불일치 frame 100% 폐기 |
| `NAV-02` | confidence·center offset·heading error를 사용한다. | 동일 bag 결과 재현 |
| `NAV-03` | stale·ambiguous·lost 입력은 throttle 0을 만든다. | fault bag에서 100% neutral |
| `NAV-04` | 출력은 normalized `VehicleCommand`다. | raw PWM 접근 0 |
| `NAV-05` | Replay에서는 I2C device와 actuator Node를 실행하지 않는다. | device mount 0 |
| `NAV-06` | Rule Drive는 `HC-M3 Simple Drive` 통과 뒤에만 허용한다. | stage 위반 시 Safety 거부 |
| `NAV-07` | throttle limit은 drive-test에서 검증한 값 이하로 둔다. | config·bag 값 일치 |
| `NAV-08` | 경로 상실 뒤 자동 재출발하지 않는다. | 운영자 reset 전 enable false |

## 6. Rule Drive 운영 범위

- 바닥이 평평하고 사람이 통제하는 짧은 코스만 사용한다.
- 장애물 없는 상태에서 실시한다.
- 운영자는 전원을 즉시 끌 수 있는 위치에서 deadman을 유지한다.
- 1회 주행은 stage config의 최대 시간과 throttle limit을 넘지 않는다.
- 성공은 tactile path 유지와 software 정지 동작으로 평가한다.

## 7. 금지 주장

- 장애물 회피·동적 정지거리
- 목표 속도 m/s 추종
- route·segment·누적거리
- odometry·위치 정확도
- 무인 운행·공공 보행로 안전성

이 규칙 기준선은 Agent를 평가하기 위한 비교 기준이며, Agent 성능이 좋아도 자동으로 폐기하지 않는다.
