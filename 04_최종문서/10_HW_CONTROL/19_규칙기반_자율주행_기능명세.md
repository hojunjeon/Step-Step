# 규칙 기반 자율주행 기능명세

## 1. 목표와 범위

AI 주행 모델 없이도 지정 루트를 안전하게 반복 주행하고 동기화된 데이터를 기록하는 평가 MVP baseline을 정의한다. 내부 영상처리·추종 수식·상태 머신 구현은 `[상세설계 단계에서 작성]`이다.

## 2. 기능 상태

| 상태 | 외부에서 관찰 가능한 요구 |
|---|---|
| `DISABLED` | actuator enable false, 출발 요청 거부 |
| `READY` | 사전 점검 통과, 아직 drive disabled |
| `MANUAL` | deadman 수동 명령만 허용 |
| `FOLLOW_ROUTE` | 점자 경로와 사전 루트로 저속 주행 |
| `AVOID` | 장애물 정지 후 허용된 경우에만 제한 회피 |
| `RECOVER` | 회피·단절 뒤 사전 루트로 복귀 |
| `STOPPED` | 목표 속도 0, 원인과 재개 조건 기록 |
| `FAULT` | latch 상태, 운영자 reset과 사전 점검 전 재개 금지 |

상태명·전이표는 상세설계에서 확정하되 위 안전 의미를 약화할 수 없다.

## 3. 주행 기능 요구

| ID | 요구사항 | 실패 시 동작 |
|---|---|---|
| `NAV-01` | 활성 route·segment·누적거리를 식별한다. | route 불일치 시 출발 금지 |
| `NAV-02` | 정상 구간에서 점자블록의 유효 경로를 추종한다. | confidence 저하 시 감속·정지 |
| `NAV-03` | 갈림길은 사전 루트 진행 방향과 일치하는 후보를 선택한다. | 불확실하면 정지·개입 요청 |
| `NAV-04` | 점자 단절은 제한 거리 동안 사전 경로·위치추정으로 통과한다. | 한계 초과 시 정지 |
| `NAV-05` | 동적 정지거리와 차체 footprint로 장애물을 판단한다. | scan stale이면 정지 |
| `NAV-06` | 회피 후 가장 가까운 유효 사전 경로로 복귀한다. | 공간·복귀 실패 시 정지 유지 |
| `NAV-07` | 수동 명령은 deadman이 있을 때만 자동 명령보다 우선한다. | 입력 단절 시 정지 |
| `NAV-08` | 모든 후보 명령은 Safety Supervisor를 통과한다. | 승인 실패 시 target 0/disabled |

- Planner·Mux·Safety의 command 우선순위
