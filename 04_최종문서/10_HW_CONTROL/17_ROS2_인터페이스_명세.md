## 2. Custom message 최소 의미

### `VehicleCommand`

sequence, 생성 timestamp, `speed_mps`, `steering_angle_rad`, enable, mode, `valid_for`를 포함한다. 단위·좌표 부호·enum·범위는 IDL 상세설계에서 고정한다.

### `ActuatorStatus`

applied sequence, 실제/추정 속도, 조향 상태, enable, fault code, control loop age를 포함한다.

### `SafetyState`

상태 enum, latched 여부, stop reason bitmask, stale input 목록, 마지막 정상 시각을 포함한다.

### `Heartbeat`

board ID, sequence, monotonic uptime, release ID, health summary를 포함한다.

## 3. TF 계약

| Parent → Child | 소유자 | 요구 |
|---|---|---|
| `map → odom` | map/route alignment | 필요할 때 하나의 broadcaster |
| `odom → base_link` | EKF | 유일한 동적 broadcaster |
| `base_link → sensor_link` | robot state publisher | calibration revision과 일치 |

## 4. 변경 관리

- interface package는 양 보드가 같은 commit·type hash를 사용한다.
- field 삭제·단위 변경·의미 변경은 breaking change다.
- QoS는 publisher·subscriber 양쪽의 contract test 대상이다.
- 실제 IDL, service/action 목록과 parameter schema는 `G0` 후속 상세설계에서 작성한다.
