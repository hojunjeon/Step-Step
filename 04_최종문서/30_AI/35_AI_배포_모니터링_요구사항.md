# AI 배포·모니터링 요구사항

> 상태: `[추후 추가 예정]` 템플릿

## 1. Artifact 요구

- model name·semantic version·immutable hash
- framework·opset·TensorRT engine compatibility
- preprocessing/postprocessing version
- training code·dataset lineage
- acceptance result와 known limitation

## 2. 배포 요구

- model과 runtime compatibility를 시작 전에 검사한다.
- model 교체는 release manifest와 rollback artifact를 함께 배포한다.
- warm-up 실패·timeout·OOM은 health와 stable error code로 노출한다.
- HW/CONTROL의 Safety·control process보다 높은 자원 우선권을 요구하지 않는다.

## 3. 모니터링 템플릿

| 영역 | 지표 |
|---|---|
| Runtime | latency, throughput, queue, error, GPU/RAM/temperature |
| Data | brightness, blur, input size, camera/calibration mismatch |
| Result | class/confidence 분포, 빈 결과, 담당자 수정률 |
| Drift | 기준 dataset·현장 slice와의 변화 `[추후 추가 예정]` |

## 4. Rollback Gate

새 모델은 acceptance dataset, Jetson 동시 부하, E2E schema 시험을 통과해야 한다. 기준 악화·runtime violation·schema mismatch 중 하나라도 발생하면 이전 artifact로 rollback한다.
