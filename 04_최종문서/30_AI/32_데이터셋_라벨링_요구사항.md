# 데이터셋·라벨링 요구사항

> 상태: `[추후 추가 예정]` 템플릿

## 1. Dataset card 필수 항목

- dataset ID·version·생성일·담당자
- 출처, 장소, 날짜, 날씨·조명, camera·calibration ID
- 수집 동의·개인정보 처리·license
- 원본 수, class 분포, route·location 분포
- train/validation/test 분할 단위와 중복 제거 방식
- 알려진 편향·제외 조건·사용 금지 목적

## 2. 라벨 계약

| 항목 | 작성할 내용 |
|---|---|
| 단위 | image, instance, polygon/mask, track, event |
| Class | 정의, 포함·제외, 우선순위, 다중 label 허용 여부 |
| Geometry | 최소 크기, occlusion, 잘린 객체, 긴 crack 병합 |
| Quality | annotator·reviewer 분리, disagreement 해결 |
| Version | guideline·tool·export schema와 변경 이력 |

## 3. Leakage 방지

같은 위치·연속 frame·같은 결함 track이 train과 test에 나뉘지 않도록 location/episode 단위 분할을 기본 후보로 한다. 최종 규칙은 `AI-M1`에서 확정한다.

## 4. Exit Criteria 템플릿

- [ ] 클래스별 최소 표본과 환경 coverage가 정의됐다.
- [ ] 라벨 검수 표본과 합의율 기준이 있다.
- [ ] split leakage 검사가 통과했다.
- [ ] 개인정보·license 검토 결과가 연결됐다.
- [ ] dataset artifact가 immutable ID로 재현된다.
