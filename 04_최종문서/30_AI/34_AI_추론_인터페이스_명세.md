# AI 추론 인터페이스 명세

## 1. 입력 계약

| 필드 | 요구 |
|---|---|
| frame identity | frame ID와 capture timestamp |
| image | encoding, width/height, camera ID |
| calibration | calibration revision |
| context | route/patrol ID, 위치·누적거리(가능할 때) |
| request | schema version, inference profile |

## 2. 출력 계약

| 필드 | 요구 |
|---|---|
| result identity | result ID, input frame ID |
| detections | class, geometry, confidence |
| model | name, semantic version, artifact hash |
| timing | queue·preprocess·inference·total latency |
| validity | valid/uncertain/error와 stable error code |

Mask·box encoding, tensor layout와 transport는 `[추후 추가 예정]`이며 HW/CONTROL·BE consumer와 contract test로 동결한다.

## 3. 실패 계약

- timeout·OOM·invalid input은 빈 정상 결과와 구분한다.
- model load 실패 시 주행 안전은 영향받지 않고 결함 기능만 degraded 상태가 된다.
- Consumer가 모르는 schema version은 결과를 격리하고 경고한다.
- 모든 result는 원본 frame과 model artifact로 역추적 가능해야 한다.

## 4. 성능 예산

목표 FPS·p99 latency·GPU/RAM/power는 주행 인지·ROS bag 동시 부하에서 `AI-M3`에 확정한다. 단독 benchmark만으로 release하지 않는다.
