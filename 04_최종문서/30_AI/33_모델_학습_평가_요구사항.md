# 모델 학습·평가 요구사항

> 상태: `[추후 추가 예정]` 템플릿. 모델 구조·학습 코드는 상세설계에서 작성한다.

## 1. 실험 재현성

모든 결과에는 code SHA, dataset version, split, preprocessing, seed, hyperparameter, framework·CUDA version과 model artifact hash를 기록한다.

## 2. 평가 항목

| 영역 | 채울 지표 |
|---|---|
| 검출·분할 | class별 precision/recall, IoU 계열, confidence calibration |
| 제품 | 유효 이벤트, 오탐, 미탐, 중복, 대표 프레임 성공 |
| 환경 slice | 장소·날짜·조명·오염·가림·camera별 성능 |
| Runtime | p50/p95/p99 latency, FPS, GPU/RAM/power |
| 안정성 | 실패 입력, timeout, 빈 결과, model load 실패 |

## 3. Acceptance dataset

RC 판정에 사용하는 immutable dataset과 class·환경 slice별 최소 기준은 `AI-M2`에서 `[확정]`한다. test set을 tuning에 반복 사용하지 않는다.

## 4. 모델 승격

`experiment → candidate → shadow → release → retired` 상태를 사용하고, 승격마다 offline 회귀·Jetson contract·E2E 이벤트 시험을 요구한다. 구체 수치는 `[추후 추가 예정]`이다.
