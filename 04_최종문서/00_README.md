# 최종 Spec 문서 안내

> 상태: `[확정]` 문서 체계 · 최종 수정일: 2026-07-20

이 폴더는 점자블록 순찰·보수계획 지원 로봇의 **최종 요구사항과 외부 계약**을 관리한다. 구현·검수 시 이 폴더를 기준으로 하며, `01_기획 문서`, `02_구체화_고도화`, `03_자율주행`은 배경·학습·설계 과정의 참고 자료다.

## 1. 문서가 다루는 수준

포함하는 내용은 제품 목표, 파트 책임, 기능·품질 요구사항, HW·실행환경 제약, ROS/API 외부 계약, 마일스톤과 인수 기준이다. 알고리즘 내부 설계, 의사코드, 클래스·함수, 완성된 launch·YAML·Dockerfile은 각 마일스톤의 상세설계 단계에서 작성한다.

## 2. 공통 상태 표기

| 상태 | 의미 |
|---|---|
| `[확정]` | 최종 기준으로 채택했으며 변경 시 영향 검토가 필요함 |
| `[검증 필요]` | 설계 기준은 있으나 실물·시험 증거가 필요함 |
| `[결정 필요]` | 대안 중 담당자 결정이 남아 있음 |
| `[추후 추가 예정]` | 해당 파트의 상세 Spec이 아직 작성되지 않음 |
| `[상세설계 단계에서 작성]` | 요구·외부 계약만 확정했고 내부 구현은 후속 산출물임 |

## 5. 통합 마일스톤

날짜보다 Exit Criteria를 우선한다.

| Gate | 통합 결과 | 통과 조건 |
|---|---|---|
| `G0 계약 동결` | 파트 책임과 인터페이스 합의 | 기능 ID, 이벤트 ID, ROS/API 소유자 확정 |
| `G1 더미 E2E` | 가짜 데이터로 전 구간 연결 | 로봇 이벤트가 BE 저장·조회까지 누락 없이 도달 |
| `G2 기본 차량` | 안전한 규칙 기반 반복 주행 | E-stop·watchdog·센서 로깅·3회 반복주행 통과 |
| `G3 기능 E2E` | 결함·장애물 이벤트 통합 | 대표 이미지·위치·모델 버전·상태가 조회됨 |
| `G4 안정화` | 고장·오프라인 복구 검증 | 재전송 누락 0, 필수 고장 주입 통과 |
| `G5 RC` | 평가용 MVP 승인 | [08 문서](./00_공통/08_통합시험_및_최종인수기준.md)의 필수 항목 통과 |

## 6. 문서 변경 규칙

1. 요구사항은 ID 없이 추가하지 않는다.
2. 파트 경계를 바꾸면 `CONTEXT.md`, 이 README, 추적표를 함께 갱신한다.
3. 목표 버전과 실장 확인 버전을 분리한다.
4. 수치 변경은 검증 근거와 영향받는 시험 ID를 기록한다.
5. 04와 기존 문서가 충돌하면 04를 기준으로 하고 충돌을 `[결정 필요]`로 남긴다.
