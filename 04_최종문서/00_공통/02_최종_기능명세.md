# 최종 기능명세

> 원칙: 기능은 외부에서 관찰 가능한 동작과 인수 조건을 정의한다. 내부 알고리즘은 `[상세설계 단계에서 작성]`이다.

## 1. 공통 형식

| 필드 | 의미 |
|---|---|
| ID | `FR-COM`, `FR-HC`, `FR-AI`, `FR-BE` 접두사 사용 |
| 입력/트리거 | 기능이 시작되는 조건 |
| 결과 | 외부에서 관찰할 수 있는 출력 |
| 실패 동작 | 오류·timeout 때의 기대 결과 |
| 인수 조건 | 시험으로 판정 가능한 기준 |

## 2. HW/CONTROL 기능

| ID | 기능 | 입력/트리거 | 결과·실패 동작 | 인수 조건 |
|---|---|---|---|---|
| `FR-HC-01` | 루트 등록·선택 | 수동 주행 기록, 운영자 선택 | 버전이 있는 루트 활성화; 유효하지 않으면 출발 금지 | 선택 루트·버전이 모든 로그에 기록 |
| `FR-HC-02` | 사전 점검 | 출발 요청 | 센서·TF·E-stop·통신 점검; 실패 시 drive disabled | 필수 health 하나라도 불량이면 출발 불가 |
| `FR-HC-03` | 규칙 기반 순찰 | 유효 루트와 센서 | 점자 경로·사전 경로를 이용한 저속 주행 | 지정 코스 3회 연속 완주 |
| `FR-HC-04` | 갈림길·단절 처리 | 점자 분기·미검출 | 사전 루트 방향 선택 또는 연결 구간 통과; 불확실하면 정지 | 지정 시나리오별 정답 진행 또는 안전 정지 |
| `FR-HC-05` | 장애물 대응 | 전방 안전영역 침범 | 정지, 이벤트 요청, 조건부 회피·복귀; 공간 부족 시 정지 유지 | 필수 장애물 시험 100% 정지 |
| `FR-HC-06` | 수동 우선 제어 | deadman이 있는 수동 명령 | 자동 명령보다 우선; deadman 해제 시 정지 | 모드 전환 중 명령 중첩 없음 |
| `FR-HC-07` | 하위 구동 제어 | 승인된 target command | 속도·조향 적용과 feedback; 만료·비정상 명령 거부 | sequence·expiry·NaN·범위 시험 통과 |
| `FR-HC-08` | 독립 안전 정지 | E-stop, heartbeat·command timeout, fault | EN 차단, fault latch, 정지 이유 기록 | 프로세스 정지와 통신 단절 시험 통과 |
| `FR-HC-09` | 동기화 로깅 | 순찰 시작 | 필수 센서·제어·상태 rosbag과 episode metadata | 2~5분 bag 10개 기록·재생 가능 |
| `FR-HC-10` | 자율주행 AI 평가 | 승인된 checkpoint·episode | Replay/Shadow 예측과 성능 기록; 모터 권한 없음 | 같은 입력으로 결과 재현, 기준선 비교표 생성 |
| `FR-HC-11` | 로컬 outbox | 서버 전송 실패 | 원본 보존·재시도·중복 방지 | 재연결 후 누락 0, 중복 허용 기준 충족 |
| `FR-HC-12` | 상태 보고 | 주행 중 주기 | 보드·센서·모드·fault·저장소 상태 제공 | stale 상태와 마지막 정상 시각 확인 가능 |
