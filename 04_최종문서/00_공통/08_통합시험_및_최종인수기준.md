# 통합시험 및 최종 인수 기준

## 1. 인수 원칙

- 평균값 하나가 아니라 성공·실패 분포와 원인을 함께 기록한다.
- 통과하지 못한 필수 안전 시험은 기능 시연 성공으로 상쇄할 수 없다.
- 시험에는 release ID, config, route, model, 장비 serial과 bag·로그 위치를 남긴다.

## 2. 필수 시나리오

| ID | 시나리오 | 합격 기준 |
|---|---|---|
| `TC-E2E-01` | 루트 선택→사전 점검→완주 | 같은 통제 코스 3회 연속 완주 |
| `TC-E2E-02` | 결함 후보 촬영→업로드→조회 | 이미지·위치·버전·신뢰도 연결 |
| `TC-E2E-03` | 장애물 접근 | 안전영역 필수 시험 100% 정지 |
| `TC-E2E-04` | 회피와 루트 복귀 | 5회 중 4회 이상 또는 안전 정지 |
| `TC-E2E-05` | 네트워크 단절·복구 | 주행 안전 유지, 재전송 누락 0 |
| `TC-E2E-06` | 동일 이벤트 재전송 | BE 중복 레코드 생성 없음 |
| `TC-E2E-07` | Replay·Shadow | AI가 actuator에 영향 없이 예측·지표 기록 |

## 3. 필수 고장 주입

| ID | 주입 고장 | 기대 결과 |
|---|---|---|
| `TC-SAFE-01` | Jetson process kill | RPi EN 차단·fault latch |
| `TC-SAFE-02` | Ethernet 단절 | command timeout 내 정지 |
| `TC-SAFE-03` | 오래된·역행·NaN 명령 | 적용 거부와 fault 기록 |
| `TC-SAFE-04` | 카메라·LiDAR·odom stale | 정의된 감속 또는 정지 |
| `TC-SAFE-05` | E-stop 입력 | ROS와 무관하게 구동 전원 차단 |
| `TC-SAFE-06` | RPi/container 재시작 | drive disabled로 복구, 수동 reset 전 구동 금지 |
| `TC-SAFE-07` | 디스크 임계치 | 대표 증거 보존, 안전 제어 영향 없음 |

## 4. 단계별 증적

| 단계 | 필수 증적 |
|---|---|
| Bench | 배선표, 전원·전류 측정, E-stop 영상 |
| SIL/Replay | bag, 예상·실제 결과, 재현 명령 |
| HIL | 바퀴를 띄운 fault injection 로그 |
| 저속 코스 | route, 개입·충돌·완주·복귀 지표 |
| E2E | API 응답, DB/event ID, 화면 캡처, outbox 로그 |

## 5. 최종 Go/No-Go

`G5 RC`는 다음이 모두 참일 때만 Go다.

- [ ] 모든 `[확정]` MVP 기능에 구현·시험 증적이 연결됐다.
- [ ] 필수 고장 주입이 모두 통과했다.
- [ ] 목표·실장 버전 차이가 해소되거나 승인된 예외로 기록됐다.
- [ ] AI·BE의 `[추후 추가 예정]` 중 시연 필수 항목이 남지 않았다.
- [ ] 알려진 위험, 운영자 개입과 시연 폴백이 승인됐다.
