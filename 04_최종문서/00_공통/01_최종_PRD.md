# 최종 PRD

> 제품: 점자블록 순찰·보수계획 지원 로봇
>
> 출시 범위: 운영자 감독하의 저속 통제 코스 평가용 MVP
>
> 상태: HW/CONTROL 구체화, AI·BE 일부 `[추후 추가 예정]`

## 1. 문제와 제품 비전

점자블록의 파손·마모·단차와 통행 장애물은 반복 현장조사가 필요하지만, 결과가 개별 사진이나 민원으로 흩어지면 구간별 변화와 보수 우선순위를 비교하기 어렵다.

> 사람이 모든 구간을 반복 조사하지 않아도 로봇의 순찰 증거를 바탕으로 점자블록 상태와 보수 후보 구간을 확인하게 한다.

## 2. 사용자

| 사용자 | 필요한 결과 |
|---|---|
| 지자체 유지관리 담당자 | 구간별 결함·장애물 근거와 보수 후보 |
| 현장 조사자 | 원본 이미지, 위치, 신뢰도와 재확인 정보 |
| 시스템 운영자 | 로봇·센서·통신 상태, 안전 정지 이유, 재처리 상태 |

시각장애인은 최종 수혜자이지만 평가 MVP를 직접 조작하는 사용자는 아니다.

## 3. 평가용 MVP 범위

### 포함

- 1~2개 사전 등록 루트 선택 및 저속 순찰
- 점자블록 추종, 단절·갈림길 처리, 장애물 정지와 제한적 회피·복귀
- 마모·크랙·파손·단차 후보의 대표 이미지와 위치 기록
- 네트워크 단절 중 로컬 저장 및 복구 후 재전송
- 서버에서 이벤트·이미지·운행 상태 조회
- E-stop, command timeout, sensor fault의 안전 정지
- 자율주행 AI의 Replay·Shadow 통합

### 제외

- 보행자 혼재 공공보도의 무인 운영
- AI가 Safety 계층을 우회하는 직접 모터 제어
- 모델 결과만으로 확정 행정 처분 또는 보수 명령 생성
- 보험·도로점용·개인정보 검토가 끝나지 않은 현장 상용 배치
- 다기기 자동 배차와 대규모 운영

## 4. 서비스 시나리오

1. 운영자가 루트를 선택하고 사전 점검을 수행한다.
2. 로봇이 점자블록과 사전 경로를 기준으로 통제 코스를 주행한다.
3. 결함 후보는 대표 이미지·위치·신뢰도와 함께 로컬에 기록된다.
4. 장애물이 안전영역을 침범하면 정지하고, 허용될 때만 저속 회피한다.
5. 서버 연결이 없으면 주행 안전 루프는 계속 동작하고 이벤트는 outbox에 남는다.
6. 연결 복구 후 BE가 이벤트를 멱등하게 수집하고 담당자가 결과를 검토한다.

## 5. 제품 원칙

- `PR-01 안전 우선`: 탐지·완주보다 사람·장비의 안전 정지가 우선이다.
- `PR-02 로컬 완결`: 서버·Wi-Fi 장애가 주행 안전에 영향을 주지 않는다.
- `PR-03 근거 보존`: 결과에는 원본 증거, 위치, 시각, 모델·설정 버전을 남긴다.
- `PR-04 사람 승인`: AI 분류와 보수 후보는 담당자가 검토·수정할 수 있다.
- `PR-05 단계적 권한`: 자율주행 AI는 Replay→Shadow→Assist Gate를 따른다.
