# 파트 간 인터페이스 명세

## 1. 소유권 원칙

- Producer는 schema와 의미·버전을, Consumer는 처리 결과와 오류를 검증한다.
- 네트워크 재시도는 같은 이벤트를 중복 생성하지 않아야 한다.
- 파트 간 계약 변경은 `G0 계약 동결` 이후 호환성 또는 공동 변경으로 처리한다.
- 안전 명령은 BE·결함 AI 인터페이스를 거치지 않는다.

## 2. HW/CONTROL → BE

| 계약 | 필수 필드 | 요구사항 |
|---|---|---|
| Patrol | `patrol_id`, `route_id`, 시작·종료, release | 재시도 가능한 고유 ID |
| DefectEvent | event UUID, 시각, 위치, route 거리, 이미지 hash, AI 결과·버전 | 원본 업로드 전 삭제 금지 |
| ObstacleEvent | event UUID, 시각, 위치, 이미지, 정지·회피 결과 | 장애물 종류 추정은 선택 |
| DeviceHealth | device ID, board/sensor 상태, fault, outbox depth | 마지막 정상 수신 시각 제공 |
| Media | content hash, mime, 크기, event UUID | metadata와 파일의 원자적 연결 필요 |

전송 프로토콜과 endpoint는 [52_로봇_BE_통신계약.md](../50_BE/52_로봇_BE_통신계약.md)에서 확정한다.

## 3. AI → HW/CONTROL

결함 AI 결과는 주행 안전 명령이 아니다.

| 필드 | 최소 계약 |
|---|---|
| `frame_id` / timestamp | 원본 프레임을 유일하게 식별 |
| detections | class, geometry, confidence |
| model identity | model name, semantic version, artifact hash |
| latency | capture-to-result 시간 |
| validity | 정상·불확실·오류 상태 |

주행 인지 모델은 HW/CONTROL 내부 계약이며 [20_자율주행_AI_단계요구사항.md](../10_HW_CONTROL/20_자율주행_AI_단계요구사항.md)를 따른다.

## 5. 오류 공통 규칙

- 오류는 안정적인 code, 사람이 읽는 message, retry 가능 여부를 포함한다.
- 시간·위치·단위가 없는 수치는 계약 위반으로 본다.
- schema version을 모르는 Consumer는 조용히 무시하지 않고 격리·경고한다.
