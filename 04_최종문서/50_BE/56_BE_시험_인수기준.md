## 1. 필수 Contract 시험

| ID | 시험 | 기대 결과 |
|---|---|---|
| `TC-BE-01` | 같은 event UUID 반복 전송 | record 중복 없음 |
| `TC-BE-02` | metadata 후 media 실패·재시도 | 고아·누락 탐지와 복구 |
| `TC-BE-03` | unknown schema version | 격리·명시적 오류 |
| `TC-BE-04` | device 인증 실패 | 저장하지 않고 감사 기록 |
| `TC-BE-05` | network flap·out-of-order | 누락 없이 최종 일관성 |
| `TC-BE-06` | 담당자 수정 | 원본·AI·수정 이력 모두 보존 |

## 2. E2E 시험

실제 robot event가 outbox→API→DB/media→조회까지 같은 ID로 연결되고, 온라인 30초 목표와 오프라인 재전송 누락 0을 검증한다.
