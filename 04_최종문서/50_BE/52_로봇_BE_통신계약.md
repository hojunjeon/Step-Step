# 로봇–BE 통신계약

## 1. 원칙

- 원격 서버에 ROS DDS를 직접 노출하지 않는다.
- Jetson gateway가 허용된 이벤트·상태·파일만 application protocol로 변환한다.
- 주행은 server response를 기다리지 않으며 실패는 local outbox로 격리한다.

## 2. 최소 계약

| 항목 | 요구 |
|---|---|
| Protocol | HTTPS 기본 후보, MQTT/WebSocket은 필요 근거 후 `[결정 필요]` |
| Identity | device ID, credential, release ID |
| Idempotency | event UUID와 idempotency key |
| Integrity | media content hash와 size |
| Version | schema version과 backward compatibility 정책 |
| Retry | exponential backoff·jitter·retryable code `[상세설계 단계에서 작성]` |
| Order | 생성 시각과 sequence를 보존하되 전역 순서를 가정하지 않음 |

## 3. 접수 상태

`accepted`, `already_exists`, `invalid`, `unauthorized`, `retry_later`, `permanent_failure`를 구분할 수 있어야 한다. 실제 HTTP status·payload는 API Spec 상세화 때 확정한다.

## 4. Outbox 계약

- metadata와 file 상태를 원자적으로 추적한다.
- 성공 응답과 server-side 조회 가능 여부를 확인하기 전 원본을 삭제하지 않는다.
- 재부팅·network flap 후에도 pending item을 복구한다.
- disk 임계치에서 대표 이벤트를 우선 보존하고 안전 제어를 방해하지 않는다.
