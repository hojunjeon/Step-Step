# 저장·보안·운영 요구사항

> 상태: `[추후 추가 예정]` 템플릿

## 1. 저장·보존

- DB record, object media, rosbag의 소유자·보존기간·삭제 절차를 구분한다.
- 원본·대표 evidence·thumbnail·파생 analysis의 보존 우선순위를 정한다.
- backup·restore의 RPO/RTO와 restore drill 기준은 `[추후 추가 예정]`이다.

## 2. 보안·개인정보

- device identity·credential rotation과 최소 권한
- TLS와 저장 암호화 범위
- human role과 원본 접근 통제
- 얼굴·번호판 blur 시점, 원본 접근·삭제기간
- audit log의 위변조 방지와 보존기간

각 항목의 정책 소유자와 승인자는 `[결정 필요]`이다.

## 3. 관측성

request/event/trace ID로 robot outbox부터 저장·분석까지 추적한다. 수집률, duplicate, retry, queue, API latency/error, DB·object 용량, media integrity와 last device contact를 모니터링한다.

## 4. 장애 대응

서버·DB·object storage·AI service 장애별 degraded behavior, alert, 복구·재처리 절차와 data reconciliation 기준은 `BE-M5` 상세설계에서 작성한다.
