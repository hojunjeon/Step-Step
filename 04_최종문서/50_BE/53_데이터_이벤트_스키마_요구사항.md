# 데이터·이벤트 스키마 요구사항

> 상태: `[추후 추가 예정]` 템플릿

## 1. 공통 필드

모든 event는 event ID, schema version, type, occurrence time, received time, device·patrol·route ID, 위치·정확도, release·config·model identity와 provenance를 가져야 한다.

## 2. 핵심 Entity

| Entity | 핵심 관계 |
|---|---|
| Device | 여러 Patrol과 Health sample 생성 |
| Route/RouteVersion | Patrol이 사용한 immutable version |
| Patrol | Event·Media·episode와 연결 |
| DefectEvent | AI result, 위치, 대표 Media와 연결 |
| ObstacleEvent | 정지·회피·복귀 결과와 Media 연결 |
| AnalysisResult | 원본 AI 결과와 server 분석을 구분 |
| Review | 담당자 수정·상태 변경 이력 |
| MaintenanceSegment | 여러 event를 묶은 파생 구간 |

## 3. 스키마 원칙

- 단위·좌표계·timezone을 명시한다.
- 원본 record를 파생 집계로 덮어쓰지 않는다.
- 삭제·보존·개인정보 상태를 media와 함께 추적한다.
- schema migration과 backward compatibility 기준은 `[추후 추가 예정]`이다.
