# BE 기능·시스템 요구사항

> 상태: `[추후 추가 예정]` 템플릿

## 1. 기능 범위

| 기능 | 외부 결과 |
|---|---|
| 이벤트 수집 | UUID 기반 멱등 접수와 안정적인 처리 상태 |
| 미디어 저장 | event와 hash로 연결된 원본 증거 |
| 순찰·기기 상태 | 최근 상태, 마지막 수신 시각, fault와 outbox depth |
| 지도·구간 조회 | 위치 이벤트와 관리 구간 결과 |
| 담당자 검토 | AI 결과 수정과 audit history |
| 보수 후보 | 근거가 연결된 후보 목록, 확정 행정 명령 아님 |

## 2. 시스템 원칙

- 같은 idempotency key는 같은 논리 결과를 반환한다.
- 원본 이벤트, AI 파생 결과와 사람 수정은 분리한다.
- 파일 업로드와 metadata 연결 실패를 탐지·복구한다.
- 서버 장애가 robot drive enable·정지 판단에 영향을 주지 않는다.
- 모든 변경은 actor, time, before/after를 감사 가능하게 남긴다.
