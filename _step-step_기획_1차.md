# 시각장애인 안전을 위한 불량 점자블록·장애물 탐지 자율주행 로봇
## 통합 작품 스펙 및 기획 문서

---

# 1. 프로젝트 개요

## 1.1 작품명

**걸음걸음**

## 1.2 한 줄 소개

점자블록 옆을 일정한 안전거리로 자율주행하면서 점자블록의 불량 상태와 보행 장애물을 탐지하고, 위치 기반 데이터를 서버에 누적하여 지자체의 보수 계획 수립을 지원하는 AIoT 시스템이다.

## 1.3 추진 배경

시각장애인은 점자블록을 이용해 이동 방향과 정지·주의 지점을 판단한다. 그러나 실제 보행 환경에서는 점자블록의 파손, 벗겨짐, 변색, 단차, 누락, 이탈과 같은 상태 이상이 발생하며, 점자블록 경로 위에 장애물이 놓이는 경우도 존재한다.

현재 점자블록 상태 조사는 사람이 직접 현장을 이동하며 확인해야 하므로 시간과 인력이 많이 필요하다. 본 프로젝트는 자율주행 로봇이 일일 또는 주간 단위로 동일 경로를 반복 순찰하면서 상태 데이터를 누적하고, 이를 지자체가 보수 우선순위와 교체 계획을 수립하는 데 활용하도록 하는 것을 목표로 한다.

---

# 2. 프로젝트 목적

본 프로젝트의 목적은 민원처럼 개별 이상 상태를 즉시 신고하고 조치하도록 하는 것이 아니다.

핵심 목적은 다음과 같다.

- 지자체의 점자블록 현장 조사 업무를 자동화·보조
- 점자블록 상태 데이터를 주기적으로 축적
- 위치별 불량 유형과 상태 변화 추적
- 위험도와 예상 교체 시기 산출
- 보수 우선순위와 공사 대상 구간 선정 지원
- 점자블록 유지관리 업무의 데이터 기반 전환
- 결과적으로 시각장애인의 보행 환경 개선 속도 향상

# 3. 핵심 서비스 시나리오

1. 관리자가 서버 또는 로봇 UI에서 미리 등록된 순찰 루트를 선택한다.
2. 로봇은 GPS 기반 글로벌 경로를 따라 이동한다.
3. 전방 카메라로 점자블록 경로를 인식한다.
4. 로봇은 점자블록 위를 직접 달리지 않고, 점자블록과 일정한 횡방향 안전거리를 유지하는 평행 경로를 주행한다.
5. 측면 상부에 설치된 YOLO 카메라가 점자블록 전체 영역을 영상으로 촬영한다.
6. YOLO 모델이 점자블록 상태를 6개 클래스로 분류한다.
7. LiDAR가 주행 경로상의 장애물을 감지하면 로봇이 정지한다.
8. 장애물 이벤트 발생 시 전방 카메라 이미지를 캡처한다.
9. 로봇은 불량 점자블록 또는 장애물 이벤트를 GPS 좌표와 함께 서버로 전송한다.
10. 장애물 이벤트 처리 후 로봇은 장애물을 회피한다.
11. 회피가 끝나면 기존 오프셋 경로로 복귀해 정상 주행을 계속한다.
12. 서버는 수신 데이터를 지도에 표시하고, VLM 및 분석 알고리즘으로 위험도와 추천 교체 시기를 산출한다.
13. 지자체 담당자는 지도와 대시보드를 통해 보수 우선순위와 보수 후보 구간을 확인한다.

# 4. 핵심 기능

## 4.1 글로벌 자율주행

- 미리 등록된 지도 경로 기반 주행
- 루트 1, 루트 2 등 복수 경로 관리
- GPS 기반 현재 위치 확인
- 현재 경로 세그먼트와 진행 방향 판단
- 점자블록이 끊기거나 갈림길이 발생하면 사전 경로 우선
- 순찰 완료 후 시작점 또는 종료점으로 이동

## 4.2 점자블록 옆 평행 주행

로봇은 점자블록 위를 직접 주행하지 않는다.

작은 바퀴가 점자블록 돌기와 단차를 반복적으로 통과하면 다음 문제가 발생할 수 있다.

- 차체 진동 증가
- 카메라 영상 흔들림
- YOLO 탐지 정확도 저하
- LiDAR·IMU 노이즈 증가
- 휠 엔코더 오차 증가
- 모터와 브래킷의 기계적 피로 증가

따라서 점자블록은 로봇이 따라가는 물리적 주행면이 아니라, 전방 카메라가 인식하는 시각적 기준선으로 사용한다.

주행 방식은 다음과 같다.

```text
점자블록 중심선 검출
→ 조감도 변환
→ 중심선의 법선 방향 계산
→ 일정 거리만큼 평행 이동
→ 로봇 목표 경로 생성
→ 목표 속도·조향각 계산
```

개념식:

```text
목표 주행 경로 = 점자블록 중심선 + 횡방향 오프셋
```

오프셋 값은 다음을 고려해 결정한다.

- 차량 중심과 YOLO 카메라 광축 간 거리
- 로봇 차체 폭
- 점자블록 폭
- 보도 가장자리와의 거리
- LiDAR가 확보한 주행 가능 공간
- 카메라 촬영 범위
- 실제 주행 오차

초기 권장 오프셋은 약 20~35cm이며, 실제 기구 조립 후 실험으로 확정한다.

## 4.3 불량 점자블록 분류

YOLO 기반 영상 분석으로 다음 6개 클래스를 분류한다.

| Class ID | 내부 코드 | 한글 분류 |
|---:|---|---|
| 0 | `BREAKAGE` | 파손 |
| 1 | `PEELING` | 벗겨짐 |
| 2 | `DISCOLORATION` | 변색 |
| 3 | `STEP` | 단차 |
| 4 | `MISSING` | 누락 |
| 5 | `DISPLACEMENT` | 이탈 |

### 클래스 정의

#### 파손

점자블록 일부가 깨지거나 구조적으로 손상된 상태

#### 벗겨짐

표면 코팅이나 재질 일부가 떨어져 나간 상태

#### 변색

정상 점자블록과 비교해 색상이 크게 바래거나 오염되어 시인성이 저하된 상태

#### 단차

인접 블록 또는 주변 보도와 높이 차이가 발생한 상태

#### 누락

일부 점자블록이 제거되거나 비어 있는 상태

#### 이탈

점자블록이 원래 위치나 정렬 방향에서 벗어나 이동·틀어진 상태

## 4.4 장애물 탐지

장애물은 LiDAR를 이용해 탐지한다.

장애물의 종류는 분류하지 않는다.

처리 흐름:

```text
LiDAR 전방 ROI 장애물 감지
→ 안전 정지
→ 전방 카메라 이미지 캡처
→ GPS 좌표와 함께 이벤트 생성
→ 서버 전송
→ 장애물 회피
→ 점자블록 오프셋 경로 복귀
```

# 5. 하드웨어 아키텍처

## 5.1 전체 구성

본 시스템은 상위 제어 보드와 하위 제어 보드로 역할을 분리한다.

```text
상위 Jetson
- 인지
- 판단
- 경로 계획
- AI 추론
- 서버 통신

하위 Raspberry Pi
- 모터 정밀 제어
- 조향 제어
- 센서 피드백
- 안전 제어
```

## 5.2 상위 보드: Jetson Orin Nano

### 주요 역할

- 전방 카메라 영상 처리
- 점자블록 중심선 검출
- 오프셋 주행 경로 생성
- YOLOv8n-seg 추론
- LiDAR 데이터 처리
- 장애물 탐지 및 회피 경로 판단
- GPS 기반 글로벌 경로 판단
- Raspberry Pi에 목표 주행 명령 전송
- 서버에 이벤트 데이터 전송
- ROS2 상위 노드 실행

### 연결 센서

| 장치 | 연결 방식 | 역할 |
|---|---|---|
| Camera 1 | USB 3.x | 전방 자율주행 영상 |
| Camera 2 | USB 3.x | 점자블록 불량 상태 분류 |
| YDLIDAR X4 Pro | USB | 장애물 탐지와 로컬 환경 인식 |
| GPS/GNSS | USB/UART | 글로벌 위치와 이벤트 좌표 |
| Raspberry Pi 5 | Ethernet | 목표 명령과 피드백 교환 |
| 서버 | Wi-Fi/Ethernet | 이벤트 업로드 |

## 5.3 Camera 1: 전방 자율주행 카메라

### 설치 위치

- 로봇 전면 중앙
- 전방과 지면이 함께 보이도록 하향 경사
- 차체 진동이 적은 위치
- 가능한 한 차체 중심선과 일치

### 역할

- 점자블록 중심선과 경계선 검출
- 점자블록 진행 방향 판단
- 갈림길 후보 검출
- 점자블록 옆 평행 오프셋 경로 생성
- 목표점 생성
- 주행 제어 판단
- 장애물 이벤트 발생 시 전방 이미지 캡처

### 실시간 제어 화면에 표시할 항목

- 점자블록 검출 영역
- 점자블록 중심선
- 평행 오프셋 목표 경로
- 현재 로봇 위치
- Look-ahead 목표점
- 목표 조향각
- 목표 속도
- 장애물 경고
- 현재 주행 모드

---

## 5.4 Camera 2: YOLO 불량 상태 분류 카메라

### 설치 위치

- 차량 측면
- 지면에서 일정 높이 확보
- 3D 프린팅 브래킷 또는 암에 장착
- 광축이 점자블록 영역을 내려다보도록 설치
- 전체 점자블록 상태가 한 화면에 들어오도록 촬영 거리 확보

### 설계 원칙

카메라가 점자블록 표면에 너무 가까우면 전체 상태를 판단하기 어렵다. 따라서 지면에서 충분히 띄워 설치하고, 일정 크기의 점자블록 구간이 영상에 들어오도록 해야 한다.

권장 초기 조건:

- 지면 높이 약 20~40cm
- 지면을 향한 하향 촬영
- 점자블록이 영상 중앙을 통과하도록 측면 위치 조정
- 영상 흔들림 방지를 위한 강성 높은 브래킷
- 필요 시 LED 조명 또는 차광 구조 추가

### 역할

- 파손 탐지
- 벗겨짐 탐지
- 변색 탐지
- 단차 후보 탐지
- 누락 탐지
- 이탈 탐지
- Bounding Box 및 Confidence 생성
- 이벤트 대표 프레임 캡처

---

## 5.5 LiDAR

### 역할

- 전방 장애물 탐지
- 로봇 주변 자유 공간 판단
- 장애물 회피 경로 생성 보조
- 점자블록 옆 주행 가능 공간 확인
- 측면 카메라 브래킷을 포함한 충돌 위험 판단
- 장애물 이벤트 트리거 생성

### 주의사항

LiDAR 장애물 판단 영역은 차량 차체만 기준으로 하면 안 된다. 측면으로 돌출된 YOLO 카메라 브래킷까지 포함한 전체 외형을 기준으로 안전 영역을 정의해야 한다.

---

## 5.6 GPS/GNSS

### 역할

- 글로벌 경로 상 현재 위치 판단
- 현재 루트 세그먼트 확인
- 갈림길에서 사전 경로 방향 선택
- 이벤트 발생 지점의 지도 좌표 생성

GPS는 정밀한 차선 유지용 센서가 아니다. 세부 주행은 전방 카메라, IMU, 휠 엔코더, LiDAR를 이용하고 GPS는 전역 경로 판단과 지도 표시용으로 사용한다.

---

## 5.7 하위 보드: Raspberry Pi 5

### 주요 역할

- Jetson으로부터 목표 속도와 목표 조향각 수신
- 구동 모터 PWM 계산
- 조향 서보 PWM 계산
- 엔코더 기반 속도 PID 제어
- IMU 데이터 수집
- 휠 엔코더 데이터 수집
- 구동계 상태 감시
- 안전 정지
- Jetson에 센서 피드백 전송

### 연결 장치

| 장치 | 연결 방식 | 역할 |
|---|---|---|
| Motor Driver HAT | GPIO/HAT | 후륜 구동 모터 제어 |
| PCA9685 | I²C | 조향 서보 PWM |
| IMU | I²C | Yaw, 각속도, 자세 |
| 휠 엔코더 | I²C/GPIO | 이동 거리와 속도 |
| ToF 센서 | UART/I²C | 단차 후보 거리 보조 |
| 비상정지 버튼 | GPIO | 하드웨어 안전 정지 |
| Jetson | Ethernet | 목표 명령과 피드백 |

## 5.8 구동계

### 후륜 구동 모터

- 목표 속도에 따라 PWM 제어
- 휠 엔코더 기반 폐루프 속도 제어
- 정지, 전진, 후진 지원

### 조향 모터

- 목표 조향각에 따라 서보 제어
- 조향 PWM과 실제 각도 간 캘리브레이션 필요
- 최대 조향각 제한 적용

# 6. 상·하위 보드 통신

## 6.1 Jetson → Raspberry Pi

Jetson은 직접 PWM 값을 보내지 않는다.

Jetson은 물리적인 목표값만 전송한다.

```yaml
topic: /vehicle/target_cmd

fields:
  stamp: timestamp
  sequence: uint32
  mode: IDLE | AUTO | AVOID | RECOVERY | STOP | MANUAL
  target_speed_mps: float
  target_steering_rad: float
  emergency_stop: bool
```

## 6.2 Raspberry Pi → Jetson

```yaml
topic: /vehicle/feedback

fields:
  stamp: timestamp
  ack_sequence: uint32
  actual_speed_mps: float
  encoder_ticks: int64
  distance_m: float
  imu_yaw_rad: float
  yaw_rate_radps: float
  motor_pwm: float
  steering_pwm_us: int
  watchdog_ok: bool
  emergency_stop_active: bool
  fault_code: string
```

## 6.3 안전 조건

- 명령이 0.5초 이상 갱신되지 않으면 정지
- Jetson 연결이 끊기면 모터 PWM 0
- Raspberry Pi 프로세스 오류 시 구동 Enable 해제
- 물리 비상정지는 소프트웨어와 독립적으로 모터 전원 차단
- AI 추론 노드와 주행 제어 노드는 분리 실행

# 7. 자율주행 알고리즘

## 7.1 기본 구조

```text
GPS 글로벌 경로
+
Camera 1 점자블록 경로
+
LiDAR 장애물 정보
+
IMU 및 엔코더 피드백
↓
Jetson 경로 판단
↓
목표 속도·조향각 생성
↓
Raspberry Pi 정밀 구동 제어
```

## 7.2 점자블록 경로 검출

1. Camera 1 영상 수신
2. 렌즈 왜곡 보정
3. 점자블록 영역 인식
4. Bird's-eye View 변환
5. 점자블록 중심선 추출
6. 곡선 피팅
7. 검출 신뢰도 계산

## 7.3 오프셋 경로 생성

```text
P_offset(s) = P_tactile(s) + d_offset × n(s)
```

- `P_tactile(s)`: 점자블록 중심선
- `n(s)`: 중심선의 좌우 법선 벡터
- `d_offset`: 목표 횡방향 거리
- `P_offset(s)`: 로봇이 따라갈 목표 경로

## 7.4 경로 추종 제어

초기 구현은 Pure Pursuit를 권장한다.

처리 순서:

```text
오프셋 경로 생성
→ Look-ahead 목표점 선택
→ 목표 곡률 계산
→ 목표 조향각 산출
→ 곡률에 따른 목표 속도 조정
```

초기 권장 속도:

| 구간 | 권장 속도 |
|---|---:|
| 직선 | 0.2~0.5m/s |
| 완만한 곡선 | 0.15~0.3m/s |
| 갈림길 | 0.1~0.2m/s |
| 장애물 접근 | 0.1m/s 이하 |
| 초기 시험 | 0.1~0.2m/s |

## 7.5 갈림길 처리

```text
Camera 1 후보 경로
+
GPS 사전 경로 방향
+
IMU Heading
↓
사전 경로와 가장 유사한 후보 선택
```

## 7.6 점자블록 단절 처리

```text
정상 검출
→ 오프셋 경로 추종

짧은 검출 실패
→ 이전 곡률과 IMU로 예측 주행

검출 실패 지속
→ GPS·IMU·엔코더 기반 글로벌 경로 추종

점자블록 재검출
→ 오프셋 경로로 점진적 복귀
```

## 7.7 장애물 회피

상태 머신 예시:

```text
NORMAL_DRIVE
→ OBSTACLE_DETECTED
→ STOP
→ CAPTURE_EVENT
→ PLAN_AVOIDANCE
→ AVOID
→ SEARCH_TACTILE_PATH
→ OFFSET_RECOVERY
→ NORMAL_DRIVE
```

## 8.4 라벨링 기준

- 한 이미지에 복수 결함이 존재하면 다중 객체 라벨링
- 파손 영역은 segmentation mask 권장
- 누락은 빈 공간 또는 블록 부재 영역 라벨링
- 이탈은 정상 정렬선과 벗어난 블록 영역 라벨링
- 변색은 정상 기준과 명확히 구별되는 영역만 라벨링
- 단차는 이미지 기반 후보 라벨이며 거리센서 근거를 병행

## 8.5 데이터 분리

- Train: 70%
- Validation: 20%
- Test: 10%

동일 장소·동일 시퀀스의 연속 프레임이 Train과 Test에 동시에 들어가지 않도록 분리한다.

## 8.6 평가 지표

- mAP50
- mAP50-95
- Precision
- Recall
- 클래스별 Confusion Matrix
- 실제 주행 영상 이벤트 검출률
- 불필요 이벤트 생성률
- 프레임 처리 속도
- Jetson 추론 지연시간

## 8.7 연속 프레임 안정화

한 프레임 결과만으로 이벤트를 생성하지 않는다.

```text
연속 N프레임 동일 클래스
+
Confidence 기준 충족
+
영상 중앙 또는 대표 촬영 영역 진입
+
객체 트래킹 상태 미촬영
↓
이벤트 생성
```

## 8.8 긴 불량 중복 처리

긴 파손이나 벗겨짐이 여러 객체로 분할될 수 있으므로 다음 후처리를 적용한다.

- Segmentation mask 병합
- 객체 간 거리 비교
- 동일 방향성 비교
- 연속 프레임 트래킹
- 구간 시작·종료 이벤트 생성
- 대표 이미지 1장 선택

## 8.9 서버 VLM 역할

YOLO가 불량 유형과 위치를 탐지하면 서버 VLM이 다음을 분석한다.

- 불량 상태 상세 설명
- 영향 면적
- 위험도 추정
- 보행 기능 저하 가능성
- 부분 보수 또는 교체 필요성
- 추천 교체 시기
- 검토 필요 여부

# 16. 최종 작품 정의

본 작품은 점자블록 위를 직접 달리는 로봇이 아니다.

정확한 작품 정의는 다음과 같다.

> 로봇은 전방 카메라로 점자블록을 시각적 기준선으로 인식하고, 점자블록에서 일정한 안전거리를 유지하는 평행 경로를 자율주행한다. 측면 상부의 YOLO 카메라는 점자블록 전체 상태를 영상으로 관찰해 파손, 벗겨짐, 변색, 단차, 누락, 이탈을 분류한다. LiDAR는 장애물을 탐지하며, 이벤트 발생 시 이미지와 GPS 좌표를 서버에 전송한다. 서버는 해당 데이터를 지도에 누적하고 위험도와 예상 교체 시기를 산출하여 지자체의 보수 계획 수립을 지원한다.
