# 걸음걸음 멀티센서 AI 주행 데이터 및 학습 전략 조사

## 0. 결론

현재 저장소가 보유한 **프로젝트 전용 AI 주행 학습 데이터는 0이다.** 점자블록 전방 영상, 카메라·LiDAR·GNSS·IMU·엔코더 동기화 로그, ROS bag, 미래 궤적 라벨, 운전자 개입 기록이 없다. `aiot-skeleton.zip` 안의 컵·마우스 분류 이미지, 차선 예제 영상, 범용 YOLO 가중치는 하드웨어 사용 예제일 뿐 이 주행 정책의 학습 데이터가 아니다.

기존 문서의 `점자블록 주행 이미지 600~1,000장`, `불량 이미지 800~1,500장` 목표는 [정지 영상 기반 인지·결함 YOLO 데이터 계획](../01_기획%20문서/04_AI_학습_및_추론_설계.md)이다. 시간 순서, 동기화 센서, 행동, 미래 궤적이 필요한 멀티센서 주행 정책에는 별개의 **episode 데이터**가 필요하다. 30 fps 영상에서 같은 1분을 잘라 얻은 1,800장은 독립적인 1,800개 주행 사례가 아니다.

따라서 지금 가능한 고도화는 다음 순서다.

1. 기존 규칙 기반 Planner와 안전 계층으로 차량을 먼저 완성하고 모든 센서를 ROS bag으로 기록한다.
2. 공개 데이터와 GNM·ViNT·NoMaD checkpoint는 일반적인 이동·시각 표현의 사전학습 prior로만 사용한다.
3. 자체 RGB+odometry 로그로 GNM 또는 ViNT를 재현하고, visual encoder를 고정한 채 작은 trajectory/mode head부터 학습한다.
4. 규칙 기반 Planner를 teacher로 사용해 정상 궤적을 자동 라벨링하고, 사람 개입 직전·복구 구간을 집중 수집한다.
5. 충분한 폐루프 검증 전에는 AI가 Planner를 대체하지 않고 shadow mode 또는 불확실 구간의 후보 궤적 제안기로만 동작한다.
6. Jetson에는 우선 FP16 TensorRT로 배포하고, 실제 target-domain calibration set이 확보된 후 INT8 PTQ, 정확도 손실이 크면 QAT 또는 teacher-student 증류를 적용한다.

**현 시점 판정:** ViNT/NoMaD 전체 fine-tuning과 AI 중심 주행은 `No-Go`, 공개 checkpoint의 offline 평가와 데이터 로거 구현은 `Go`다.

## 1. 조사 범위와 증거 수준

본 문서는 논문, 저자 프로젝트 페이지, 공식 데이터셋 페이지, 공식 코드, NVIDIA 문서만 근거로 사용했다. 아래에서 `engineering estimate`로 표시한 수집량과 합격선은 논문이 보장하는 숫자가 아니라, 이 프로젝트의 저속 주행·일정·센서 구성에 맞춘 의사결정 기준이다.

공개 논문의 성공률이나 데이터 시간이 곧 걸음걸음의 성능을 보장하지 않는다. 특히 공개 데이터와 자체 차량 사이에는 국가별 점자블록 규격, 카메라 높이·화각, Ackermann 조향, 속도, 센서 종류, 보도 구조의 차이가 있다.

## 2. 현재 데이터 감사

### 2.1 실제 보유량

| 데이터 | 현재 수량 | 품질 판정 | 주행 정책 사용 가능성 |
|---|---:|---|---|
| 프로젝트 전용 전방 주행 영상 | 0 | 없음 | 불가 |
| 동기화 RGB·LiDAR·GNSS·IMU·encoder 로그 | 0 episode | 없음 | 불가 |
| ROS bag | 0 | 없음 | 불가 |
| 미래 waypoint·trajectory 라벨 | 0 | 없음 | 불가 |
| mode·stop·intervention 라벨 | 0 | 없음 | 불가 |
| 독립 route/date/site test episode | 0 | 없음 | 불가 |
| 컵·마우스·차선 예제와 범용 YOLO 가중치 | 일부 | 프로젝트 외 예제 | 학습 데이터로 부적합 |

따라서 현재 데이터의 양과 질을 “적다”고 평가할 단계가 아니라 **아직 측정할 대상이 없다**고 평가해야 한다.

### 2.2 기존 이미지 목표와 주행 episode의 차이

| 항목 | 결함·인지 데이터 | AI 주행 정책 데이터 |
|---|---|---|
| 기본 단위 | 서로 다른 장면의 이미지·mask | 시작부터 종료까지 이어진 주행 episode |
| 필수 입력 | RGB, 결함 class/mask | 시간 순 RGB + 센서 + 이전 행동 + route goal |
| 정답 | class, box, mask | 미래 위치·yaw, mode, stop, 개입·복구 |
| 중복 판단 | 같은 결함 연속 프레임은 중복 | 시간 연속성이 필수이나 같은 route 반복은 강한 상관 |
| split | 물리 위치·결함 단위 | route·날짜·장소·주행 회차 단위 |
| 공개 데이터 대체성 | 점자블록 segmentation에 일부 가능 | 자체 차량·센서·제어 데이터가 반드시 필요 |

## 3. GNM·ViNT·NoMaD가 실제로 요구하는 데이터

### 3.1 공통 데이터 표현

공식 `visualnav-transformer` 코드는 custom dataset을 `시간 순서가 있는 전방 RGB 이미지 + 같은 시점의 2D position/yaw odometry`로 정의하고, 기존 checkpoint에서 이어 학습하는 절차를 제공한다. 즉 원래 모델은 raw LiDAR·GNSS·IMU를 직접 받지 않는다. 멀티센서 fusion은 이 프로젝트가 새로 설계하고 학습해야 하는 부분이다. [공식 저장소의 데이터 형식과 fine-tuning 절차](https://github.com/robodhruv/visualnav-transformer#training)

GNM 논문은 서로 다른 로봇의 action을 공유하기 위해 local odometry로 만든 상대 waypoint와 yaw를 사용하고, 시간 문맥과 정규화된 action space가 일반화에 중요하다고 보고했다. [GNM 논문](https://arxiv.org/abs/2210.03370)

### 3.2 모델별 판단

| 모델 | 원 연구의 데이터·출력 | 자체 데이터 요구 | 걸음걸음에서의 위치 |
|---|---|---|---|
| GNM | 6개 로봇의 60시간 trajectory, temporal context, 상대 waypoint·yaw | RGB+odometry 형식 변환, 카메라·차량 적응 필요 | 가장 단순한 재현·경량 baseline |
| ViNT | 여러 로봇·환경의 수백 시간 규모 사전학습, image goal과 waypoint/distance; GPS나 route command token으로 task adaptation 가능 | 소량 적응이 가능하다고 보고하지만 수량 보장은 없음; 새 멀티센서 encoder는 자체 학습 필요 | 주행 prior와 visual encoder 후보 |
| NoMaD | 다중 ground robot 데이터에 goal masking과 diffusion action decoder를 학습, goal-directed와 exploration 통합 | 프로젝트의 희귀 상황과 다중 안전 궤적 분포가 있어야 이점 검증 가능 | 데이터와 런타임이 확보된 뒤의 후속 연구 |

GNM은 60시간의 heterogeneous trajectory가 단일 도메인 정책보다 새 로봇에 잘 일반화했다고 보고했다. 다만 이는 점자블록 offset 유지나 한국 보도 안전을 검증한 결과가 아니다. [GNM 논문](https://arxiv.org/abs/2210.03370)

ViNT는 사전학습 모델이 scratch 학습보다 작은 downstream dataset으로 적응할 수 있고, goal encoder를 GPS waypoint나 routing command 표현으로 교체할 수 있음을 보였다. 논문은 이를 “small amount”로 표현하지만 걸음걸음에 필요한 최소 시간이나 episode 수를 제시하지 않는다. [ViNT 논문](https://arxiv.org/abs/2306.14846)

NoMaD는 ViNT 계열 표현과 diffusion decoder를 사용해 goal-conditioned navigation과 goal-agnostic exploration을 하나로 학습한다. 공식 설정은 4개 관측 프레임 문맥, 8개 waypoint 예측, 10 diffusion iteration을 사용한다. 이 반복 추론과 다중 궤적의 이점은 Orin Nano에서 다른 YOLO 작업과 동시에 실측해야 한다. [NoMaD 논문](https://arxiv.org/abs/2310.07896), [공식 NoMaD 설정](https://github.com/robodhruv/visualnav-transformer/blob/main/train/config/nomad.yaml)

### 3.3 중요한 한계

공개 checkpoint를 그대로 사용하는 것은 `zero-shot baseline`이지 학습 완료가 아니다. 다음 항목은 공개 모델에 없다.

- 점자블록에서 20~35 cm offset을 유지하는 목표
- 한국 보도의 선형·점형 블록 전환과 단절 후 재합류
- 이 차량의 카메라 높이·왜곡·Ackermann 최소 회전반경
- LiDAR·GNSS·IMU·encoder fusion과 센서 dropout
- 프로젝트의 `FOLLOW_TACTILE`, `OCCLUDED`, `CROSSWALK`, `REJOIN`, `STOP` mode
- 독립 Safety Supervisor의 승인·거부 결과

### 4.2 공개 데이터 조합 권고

**정책 사전학습:** ViNT/NoMaD 공식 파이프라인이 이미 지원하는 RECON, GO Stanford, TartanDrive, SACSoN 계열을 사용한다. 공식 config도 이 mixture를 명시한다. [ViNT 공식 설정](https://github.com/robodhruv/visualnav-transformer/blob/main/train/config/vint.yaml)

**보도·사람·장애물 표현:** SCAND와 CODa를 보조 representation 학습 또는 perception head에 사용한다.

**점자블록 표현:** GuideTWSI와 SideSeeing을 segmentation 또는 보조 tactile-visible head에 사용한다. GuideTWSI는 합성 데이터 추가가 모든 평가 모델의 segmentation 지표를 개선했다고 보고했으므로 synthetic augmentation의 직접 근거가 있다. 다만 이는 trajectory 정책 개선 근거는 아니다. [GuideTWSI 공식 결과](https://guidedogrobot-tactile.github.io/)

**라이선스 주의:** GO Stanford는 공식 페이지상 CC BY-NC-SA 3.0이므로 상용화 단계에서는 사용 범위를 다시 검토해야 한다. 데이터 다운로드 전 각 데이터셋의 최신 라이선스와 재배포 조건을 별도 기록한다. [GO Stanford 라이선스](https://svl.stanford.edu/projects/gonet/dataset/)

## 5. 자체 데이터 수집 설계

### 5.1 한 episode에 반드시 저장할 것

```yaml
episode:
  route_id: ROUTE_01
  site_id: SITE_A
  run_id: 20260719_153000
  weather: clear
  lighting: afternoon
  control_source: MANUAL | RULE_PLANNER | AI_SHADOW | AI_ASSIST

time_aligned_sample:
  timestamp_ns: 0
  front_rgb: image
  lidar_scan: ranges
  gnss: [lat, lon, covariance]
  imu: [yaw, yaw_rate, accel]
  wheel_odom: [x, y, yaw, speed]
  command: [speed, steering]
  actual_actuation: [speed, steering]
  route_goal_base_link: [x, y, heading]
  tactile_mask_or_centerline: optional
  safety_override: false
  human_intervention: false
```

미래 waypoint는 수동으로 모든 프레임에 그리지 않고, timestamp가 맞는 odometry에서 `t+0.5 ... t+2.5 s`의 상대 pose를 자동 생성한다. 사람이 직접 확인할 라벨은 mode, stop, 센서 이상, 개입 시작·종료, 개입 이유, 복구 성공 여부에 집중한다. GNM도 상대 waypoint label을 local odometry에서 얻는다. [GNM 논문](https://arxiv.org/abs/2210.03370)

### 5.2 품질 gate

아래 수치는 이 프로젝트용 `engineering estimate`다.

| 검사 | Go 기준 | No-Go 예시 |
|---|---|---|
| 센서 시간 정렬 | RGB 기준 95% 이상 sample에서 최근 센서가 ±50 ms 이내 | topic별 clock가 다르거나 timestamp 역행 |
| frame·scan 손실 | 핵심 topic 누락률 1% 이하, 누락 mask 저장 | 누락을 0으로 채우고 정상값처럼 사용 |
| camera calibration | intrinsic/extrinsic version 고정·기록 | 카메라 각도 변경 후 같은 dataset으로 취급 |
| odometry | 정지·직선·회전 calibration test 통과, pose jump 자동 탐지 | 미래 trajectory label이 흔들리거나 좌표계 불명 |
| action | 명령값과 실제 encoder 반응을 둘 다 기록 | PWM 명령만 있고 차량 반응 없음 |
| episode metadata | route/site/date/weather/control source 완비율 100% | frame 파일명만 있고 주행 회차 식별 불가 |
| split | site·route·date·episode 단위 분리 | 같은 영상 인접 frame이 train/test에 동시 존재 |
| safety | E-stop·watchdog·LiDAR stop은 학습 모델과 독립 | AI 출력을 바로 PWM으로 전달 |

### 5.3 데이터 규모별 Go/No-Go

아래 수량은 연구 논문의 보장값이 아닌 `engineering estimate`다. 총 frame보다 **독립 episode, 장소, 날짜, 희귀 상황 수**를 우선한다.

| 단계 | 자체 데이터 목표 | 허용되는 결론 | 판정 |
|---|---|---|---|
| D0 | 0시간, 0 episode | 공개 checkpoint 실행·로거 개발만 가능 | AI fine-tuning No-Go |
| D1 로거 검증 | 0.5~1시간, 10회 이상 짧은 run, 2~3 route | 동기화·좌표·라벨 자동생성 검증 | 학습 성능 주장 No-Go |
| D2 최소 prototype | 5~10시간, 50~100 episode, 3개 이상 site/날짜 조건 | frozen encoder + 작은 head, GNM/ViNT visual baseline, shadow mode | 제한적 Go |
| D3 assist pilot | 15~25시간, 150~300 episode, 5개 이상 site, 핵심 mode별 50개 이상 독립 event | 불확실 구간 후보 궤적·mode 분류를 safety gate 뒤에서 시험 | 조건부 Go |
| D4 planner 대체 검토 | 50시간 이상, 계절·날씨·보행자·희귀 실패를 포함한 수백 episode와 충분한 intervention | 통제 코스에서 제한적 대체 A/B 검토 | 프로젝트 MVP에는 No-Go |

추가 gate:

- `OCCLUDED`, `CROSSWALK`, `AVOID`, `REJOIN`, `STOP` 각각에 train/validation/test의 독립 episode가 없으면 해당 mode 자동 실행은 No-Go다.
- 안전 관련 test는 “몇 frame 맞음”이 아니라 독립 시나리오 반복 횟수와 충돌·개입·최소 clearance로 판정한다.
- D2의 5~10시간도 전 구간 end-to-end 정책을 새로 학습하기에는 작다. pretrained encoder 고정, 작은 head, 제한된 행동 범위라는 조건에서만 의미가 있다.

### 5.4 현실적인 수집 구성

`engineering estimate` 권장안:

- 정상 점자블록 추종 40%
- 부분 가림·단절·조명 변화 20%
- 갈림길·횡단보도·재합류 15%
- 정적 장애물 회피·정지 15%
- 센서 dropout·GNSS 불량·blur·노출 이상 10%

각 route는 수동 expert, 규칙 기반 Planner, AI shadow를 같은 조건에서 반복한다. 정상 직선만 오래 주행해 시간을 채우지 말고, 실패 직전과 복구를 독립 event로 색인한다.

## 6. 적은 현장 데이터에서 가능한 학습 방법

### 6.1 1순위: pretrained encoder 고정 + 작은 adapter/head

ViNT의 전체 network를 바로 fine-tuning하지 않는다.

1. GNM 또는 ViNT checkpoint의 RGB encoder를 고정한다.
2. route-goal, IMU/odometry history, LiDAR polar feature를 작은 MLP/token encoder로 만든다.
3. fusion layer와 mode·trajectory·stop head만 학습한다.
4. validation이 개선될 때 encoder 상위 block만 낮은 learning rate로 해제한다.

ViNT는 downstream goal modality를 새 goal token으로 바꾸는 prompt-tuning식 적응을 제시하고, 공식 코드는 checkpoint에서 이어 학습을 지원한다. [ViNT 논문](https://arxiv.org/abs/2306.14846), [공식 fine-tuning 절차](https://github.com/robodhruv/visualnav-transformer#training)

이 방식은 “encoder가 점자블록을 이미 안다”는 뜻이 아니다. GuideTWSI 등으로 tactile-visible/segmentation auxiliary head를 먼저 적응시키고, 주행 head는 자체 trajectory로 학습한다.

### 6.2 2순위: 규칙 기반 Planner teacher → 작은 student

규칙 기반 점자블록 중심선·사전 경로·LiDAR collision checker가 정상 구간에서 생성한 안전 waypoint를 teacher label로 저장한다. student는 RGB·route·sensor history에서 teacher의 trajectory, mode, stop score를 모방한다.

권장 loss:

```text
L = L_expert_trajectory
  + 0.5 * L_teacher_trajectory
  + L_mode
  + 2.0 * L_stop
  + L_feature_distillation
```

Knowledge distillation은 큰 model/ensemble의 soft target을 작은 model에 전달해 inference 비용을 줄이는 방법이다. 그러나 teacher가 틀리거나 관측하지 않은 상태를 자동으로 해결하지는 않는다. [원 Knowledge Distillation 논문](https://arxiv.org/abs/1503.02531)

더 직접적인 최근 navigation 사례로 MBRA는 짧은 horizon의 model-based expert가 noisy/passive trajectory를 재라벨링하고, 그 결과를 long-horizon visual/GPS navigation policy에 distill한다. 이 프로젝트도 “기존 Planner를 teacher로 삼아 수집 로그를 재라벨링”하는 구조를 적용할 수 있다. [Learning to Drive Anywhere via Model-Based Reannotation 프로젝트](https://model-base-reannotation.github.io/), [논문](https://arxiv.org/abs/2505.05592)

**한계:** teacher-student는 데이터 수집을 없애지 않는다. 같은 카메라·센서로 다양한 상태를 지나간 로그가 있어야 teacher label을 만들 수 있으며, teacher가 처리하지 못하는 가림·잘못된 갈림길·복구에는 사람 개입 label이 필요하다.

### 6.3 3순위: intervention learning과 DAgger

offline behavior cloning은 expert가 방문한 정상 상태만 보므로, 작은 조향 오차가 누적되어 학습 분포 밖으로 벗어날 수 있다. DAgger는 현재 policy가 방문한 상태를 expert가 다시 라벨링해 dataset에 합치는 방식이다. [DAgger 원 논문](https://proceedings.mlr.press/v15/ross11a.html)

실차에서는 모든 state를 사람이 라벨링하는 DAgger보다 다음 intervention loop가 현실적이다.

```text
규칙 기반/AI shadow 주행
→ 사람이 위험·오판 직전에 takeover
→ takeover 전 2~3초 + 교정 + 복구 3~5초 저장
→ intervention reason과 expert trajectory 생성
→ 기존 정상 데이터와 섞어 재학습
```

Learning from Interventions는 expert 개입과 비개입 모두 현재 상태·행동 품질에 대한 정보를 준다고 설명하며, 전 state 라벨링 부담을 줄이는 방법을 제안한다. [RSS 2020 공식 논문 페이지](https://roboticsconference.org/2020/program/papers/55.html)

실차 DAgger는 반드시 통제 코스, 저속, 즉시 takeover, 독립 E-stop에서만 수행한다. 공공 보도에서 online exploration으로 데이터를 모으지 않는다.

### 6.4 4순위: 무라벨 영상·센서 자기지도

라벨이 적어도 모든 순찰 RGB와 sensor stream을 버리지 않는다.

- 가까운 시점 frame의 표현 일관성
- 다음 frame/latent 또는 optical flow 예측
- image와 odometry displacement의 정합
- RGB와 LiDAR free-space의 cross-modal matching
- sensor masking 후 누락 modality 복원
- 같은 장소의 다른 날짜·조명 positive pair

ViNT 자체도 robot trajectory에서 image goal과 상대 pose를 자동 구성하는 self-supervised goal-reaching objective를 사용한다. [ViNT 논문](https://arxiv.org/abs/2306.14846)

DINOv2는 라벨 없이 학습한 범용 visual feature와 작은 distilled ViT를 공개한다. 점자블록 segmentation·보도 appearance encoder 비교군으로는 유용하지만, ViNT 연구는 일반 ImageNet/VC-1 표현만으로는 어려운 navigation affordance에 충분하지 않았다고 보고한다. 따라서 DINOv2는 **인지 보조 encoder**, ViNT는 **navigation-aligned baseline**으로 비교한다. [DINOv2 논문](https://arxiv.org/abs/2304.07193), [ViNT 논문](https://arxiv.org/abs/2306.14846)

### 6.5 5순위: 합성 데이터와 시뮬레이션

GuideTWSI는 UE4/AirSim으로 재질·조명·날씨·camera viewpoint를 바꾼 15K 이상의 synthetic TWSI RGB/depth/box/mask를 만들고, real data에 추가했을 때 여러 segmentation model의 성능을 개선했다. 이는 점자블록 인지용 synthetic data의 강한 근거다. [GuideTWSI 공식 프로젝트](https://guidedogrobot-tactile.github.io/)

적용 범위:

- 점자블록 색·재질·마모·그림자·물기·가림 segmentation
- camera pitch·height·exposure 변화
- 센서 dropout과 timestamp jitter fault injection
- 장애물 배치별 collision checker와 stop head 검증
- 실제 수집하기 위험한 실패 scenario의 regression test

시뮬레이터의 action을 그대로 실제 주행 teacher로 쓰는 것은 후순위다. 보도 마찰, 조향 backlash, GNSS multipath, 점자블록 설치 오차가 모델링되지 않으면 sim-to-real gap이 남는다. Domain randomization은 simulator의 visual·dynamics parameter를 넓게 변화시켜 transfer를 돕지만 target-domain real validation을 대체하지 않는다. [Domain Randomization 원 논문](https://arxiv.org/abs/1703.06907), [Habitat-Sim 공식 저장소](https://github.com/facebookresearch/habitat-sim)

### 6.6 6순위: domain adaptation

낮은 위험부터 적용한다.

1. 자체 camera의 color/exposure/blur/occlusion augmentation
2. public:own sampling ratio 조절과 own data oversampling
3. source와 target의 별도 normalization statistic
4. target 무라벨 영상에 teacher-student consistency
5. 충분한 검증셋이 있을 때만 domain-adversarial feature alignment

DANN은 labeled source와 unlabeled target에서 domain을 구분하기 어려운 feature를 학습하는 방법이다. [JMLR 원 논문](https://jmlr.org/papers/v17/15-239.html) 그러나 안전 정책에서는 target feature를 맞췄다는 사실만으로 action 정답이 보장되지 않는다. 최종 선택은 자체 target-domain closed-loop test로 한다.

## 7. 증류 대안 평가

### 7.1 가능한 teacher 종류

| teacher | student가 배울 것 | 장점 | 위험 | 권장도 |
|---|---|---|---|---|
| 규칙 기반 Planner + collision checker | waypoint, stop, feasible mask | 이미 구현할 시스템을 활용, 설명 가능 | 규칙의 한계를 그대로 복제 | 매우 높음 |
| ViNT fine-tuned model/ensemble | trajectory distribution, feature | 다양한 visual prior | 자체 데이터가 먼저 필요 | 중간 |
| NoMaD | 복수 궤적 또는 diffusion noise prediction | 다중 선택 표현 | 반복 sampling 비용, distillation 복잡도 | 후속 |
| offline VLM/대형 vision teacher | mode·scene attribute·hard negative | 라벨 비용 절감 | 행동 teacher가 아니며 hallucination 검수 필요 | 인지 보조만 |
| simulator privileged expert | collision-free trajectory | 희귀 scenario 대량 생성 | sim-to-real gap | 보조 |

### 7.2 추천 student

Jetson student는 다음 정도로 제한한다.

```text
4-frame RGB at 96~160 px
→ EfficientNet-B0/MobileNet급 encoder
+ IMU/odometry history MLP
+ LiDAR polar histogram MLP
+ route-goal token
→ 2~4 layer compact fusion
→ mode + 5~8 waypoint + stop + uncertainty
```

NoMaD를 사용해야 한다면 먼저 diffusion sampling step을 줄인 teacher를 offline에서 만들고, 여러 sampled trajectory의 평균이 아니라 safety cost로 선택된 trajectory를 deterministic student에 distill한다. Progressive Distillation은 diffusion sampling step을 반복적으로 절반으로 줄이는 원리를 제시하지만 이미지 생성 benchmark의 결과이므로, navigation action에서는 별도 폐루프 검증이 필요하다. [Progressive Distillation 논문](https://arxiv.org/abs/2202.00512)

### 7.3 증류 Go/No-Go

- teacher가 자체 held-out episode에서 규칙 baseline보다 낫지 않으면 distillation `No-Go`.
- teacher와 student가 같은 train/test split에 과적합했다면 `No-Go`.
- student가 open-loop ADE만 좋아지고 intervention/km, collision, stop recall이 나빠지면 `No-Go`.
- student가 FP16 TensorRT에서 이미 latency·memory budget을 만족하면 복잡한 KD를 추가하지 않는다.
- KD 후 worst-case mode recall 또는 calibration이 허용치를 벗어나면 larger student 또는 mixed precision으로 되돌린다.

## 8. Jetson Orin Nano 배포 전략

Jetson Orin Nano 8GB는 공식 사양상 8GB LPDDR5와 Ampere GPU를 사용한다. 최신 소프트웨어의 Super 모드는 최대 67 TOPS를 제시하지만 실제 성능은 전력 모드, thermal, JetPack/TensorRT 버전, 동시에 실행하는 YOLO·ROS 2·영상 기록에 좌우된다. [NVIDIA Jetson Orin Nano 공식 사양](https://developer.nvidia.com/embedded/jetson-modules)

Orin Nano에는 DLA가 없으므로 “YOLO는 DLA, 주행 모델은 GPU” 같은 분리는 불가능하다. GPU 공유를 전제로 측정해야 한다. NVIDIA의 DLA 지원 목록은 Orin NX·AGX Orin을 포함하지만 Orin Nano를 포함하지 않는다. [NVIDIA DLA 공식 안내](https://developer.nvidia.com/blog/getting-started-with-the-deep-learning-accelerator-on-nvidia-jetson-orin/)

### 8.1 적용 순서

1. PyTorch FP32 reference에서 정확도 고정
2. ONNX export와 operator 지원 확인
3. TensorRT FP16 engine 생성
4. 실제 camera+sensor replay와 YOLO 동시 실행으로 p50/p95/p99 latency, RAM, GPU, 온도, 전력 측정
5. target-domain representative episode로 INT8 PTQ
6. mode/stop/trajectory 성능이 나빠지면 QAT 또는 일부 layer FP16 유지
7. 그래도 예산을 넘을 때 encoder 축소·frame 수 감소·KD 적용

TensorRT는 ONNX 등에서 engine을 만들고 mixed precision과 quantization을 지원한다. Reduced precision은 memory와 속도를 개선할 수 있지만 rounding·clamping과 FP16 overflow로 정확도가 저하될 수 있으므로 실제 데이터로 검증해야 한다. [TensorRT 공식 문서](https://docs.nvidia.com/deeplearning/tensorrt/latest/), [정확도 고려사항](https://docs.nvidia.com/deeplearning/tensorrt/latest/inference-library/accuracy-considerations.html)

### 8.2 대안 비교

| 방법 | 추가 학습 | 기대 효과 | 데이터 요구 | 권장 순서 |
|---|---:|---|---|---:|
| TensorRT FP16 | 없음 | 가장 낮은 구현 위험, memory·latency 절감 | replay validation set | 1 |
| INT8 PTQ | 없음 | 추가 memory·latency 절감 가능 | 실제 분포 calibration set 필수 | 2 |
| INT8 QAT | 필요 | PTQ보다 정확도 보존 가능 | train+validation episode | 3 |
| 구조 축소 | 재학습 | frame/encoder/fusion 비용 직접 감소 | 자체 데이터 | 3 |
| teacher-student KD | 재학습 | 작은 model의 정확도 보완 | 좋은 teacher와 동일 분포 로그 | 4 |
| NoMaD diffusion distillation | 큰 추가 연구 | sampling step 감소 가능 | 다중 trajectory와 폐루프 test | 5 |

INT8 calibration set은 정상 직선 frame만 뽑지 않고 각 mode, 밝기, blur, 가림, 센서 범위를 대표해야 한다. TensorRT 공식 문서도 calibration data가 전체 inference data를 대표해야 한다고 명시한다. [TensorRT quantization 문서](https://docs.nvidia.com/deeplearning/tensorrt/10.x.x/inference-library/work-quantized-types.html)

### 8.3 배포 합격선

다음은 `engineering estimate`다.

- 주행 policy 단독 p99 inference ≤ 100 ms
- YOLO·ROS 2·영상 기록 동시 실행 시 전체 decision p99 ≤ 150 ms
- 30분 이상 thermal soak에서 timeout 0회
- GPU/RAM 여유 20% 이상
- FP32 대비 FP16/INT8의 mode macro-F1, stop recall, ADE/FDE, uncertainty calibration을 모두 비교
- 어떤 정밀도에서도 Safety Supervisor 주기는 AI inference와 독립 유지

속도가 0.3 m/s라면 150 ms 동안 4.5 cm 이동하므로 latency는 단순 FPS가 아니라 sensing-to-command와 braking distance에 포함해 평가한다.

## 9. 권장 실험 순서

### Phase A — 1주: 데이터 기반 만들기

- 모든 topic의 timestamp·frame_id·calibration version을 포함한 ROS bag logger
- 수동 조종, 규칙 Planner, E-stop/intervention event 기록
- 자동 trajectory label 생성과 episode validator
- D1 0.5~1시간 로그에서 품질 gate 통과

### Phase B — 1주: 공개 baseline

- GNM과 ViNT checkpoint를 동일 replay에 실행
- 공개 데이터 zero-shot, 자체 topomap, 규칙 Planner를 비교
- Jetson FP16 latency와 YOLO 동시 실행 측정
- 성능과 무관하게 AI는 shadow mode 유지

### Phase C — 2주: 최소 학습

- D2 5~10시간 자체 episode 수집
- frozen ViNT encoder + route/IMU/odometry 작은 head
- 규칙 Planner teacher label + 사람 intervention label
- route/date/site split의 open-loop 평가

### Phase D — 이후: 제한적 assist

- 안전 요원 동반 통제 코스에서 AI candidate를 Safety Supervisor 뒤에 연결
- intervention/km, route completion, offset error, stop recall, rejoin success 측정
- 실패 직전·복구 episode를 다시 학습에 포함
- D3와 폐루프 gate를 통과할 때만 특정 mode 자동 개입

### Phase E — 후속 연구

- NoMaD multi-trajectory
- diffusion step 축소 또는 deterministic student distillation
- synthetic fault/rare scenario 확대
- 더 많은 site·날짜·계절로 D4 검토

> **최종 권고:** 현재 부족한 것은 모델 선택이 아니라 자체 차량에서 수집한 고품질 episode다. 증류·자기지도·합성·domain adaptation은 이 episode의 효율을 높일 수 있지만 대체할 수는 없다. 먼저 5~10시간의 동기화된 자체 로그로 frozen ViNT/경량 head와 규칙 teacher를 검증하고, 15~25시간과 희귀 mode별 독립 event가 쌓인 뒤에만 AI assist를 폐루프로 평가한다. NoMaD와 diffusion distillation은 그 이후 단계로 둔다.
