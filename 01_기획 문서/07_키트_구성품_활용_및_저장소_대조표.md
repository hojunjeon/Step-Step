### 입력 자료

- 첨부 엑셀

- 제공 저장소  
  `https://github.com/hojunjeon/AIoT_Skeleton`

### 확인 제약

문서 작성 시점에는 제공 저장소의 내부 파일을 비로그인 공개 환경에서 확인하지 못했다. 따라서 다음을 임의로 가정하지 않았다.

- 기존 ROS 2 package 이름
- Motor Driver HAT 제어 IC와 핀맵
- Raspberry Pi GPIO 라이브러리
- Jetson·Pi 통신 방식
- 카메라·LiDAR launch 파일
- 기존 route/navigation 구현
- Docker·서버 템플릿 존재 여부

팀 계정으로 저장소에 접근한 뒤 본 문서의 `저장소 대조 절차`를 수행해야 한다.

## 2. Jetson 점검표 반영

첨부된 Jetson 체크리스트에서 다음 항목을 확인했다.

- microSD: 정상 표시
- 40핀 pin 1의 3.3V: 불량 표시
- 40핀 pin 2의 5V: 불량 표시
- I²C1 SDA: 정상 표시
- 나머지 핀은 공란이 많음

### 설계 반영

1. Jetson 40핀 헤더를 핵심 전원원으로 사용하지 않음
2. 카메라·LiDAR·GNSS는 USB
3. 저수준 GPIO/I²C는 Raspberry Pi
4. Jetson↔Pi는 Ethernet
5. Jetson 헤더 재시험 전 직접 배선 금지
6. 재시험 결과는 사진·멀티미터 값과 함께 기록

### 재시험 절차

- Jetson 전원 OFF 상태에서 핀 단락 확인
- 공식 pinout과 핀 번호 대조
- 전원 ON 후 무부하 전압 측정
- known-good GND 사용
- 3.3V/5V 전압 측정
- 부하 연결 없이 먼저 확인
- 결과가 여전히 이상하면 헤더 사용 금지

## 8. 재사용 판단 기준

## 8.1 REUSE

- 현재 지급품에서 실행됨
- Ubuntu 22.04/ROS 2 Humble 호환
- 핀맵과 전원 방식이 확인됨
- 안전 정지 가능
- license 문제 없음
- 팀원이 이해하고 디버깅 가능

## 8.2 ADAPT

- 핵심 로직은 유효하나 topic/API 이름이 다름
- Pi 4 GPIO를 Pi 5 방식으로 수정
- 카메라·LiDAR device path만 수정
- Python package 버전 업데이트 필요
- ROS 1 코드를 ROS 2로 일부 변환

## 8.3 REPLACE

- 하드코딩된 위험 PWM
- watchdog 없음
- 전원·핀맵이 지급품과 다름
- 코드 실행 환경을 재현할 수 없음
- 장애물 정지보다 AI 추론이 우선되는 구조
- 라이선스·출처 불명
- 2일 이상 디버깅해도 핵심 기능이 실행되지 않음

기존 코드가 있다는 이유로 무조건 재사용하지 않는다. 반대로 명칭과 스타일이 다르다는 이유만으로 재작성하지 않는다.

## 11. 결론

지급 키트는 차체, Jetson, Raspberry Pi, 카메라, LiDAR, 모터, 서보, 모터 드라이버, 화면과 조이스틱까지 제공하므로 핵심 로봇 플랫폼을 새로 구매할 필요가 없다.

사비 구매는 다음에 집중한다.

1. 위치·자세·이동거리 센서
2. 단차 보조센서
3. 이동 전원
4. 비상정지·퓨즈
5. 안정적인 전원 변환과 배선

저장소 코드의 실제 내용이 확인되면 본 문서의 구조를 그대로 강제하지 말고, 검증된 기존 기능은 재사용한다. 단, **안전 정지, 전원 분리, 이벤트 멱등성, 2주차 인터페이스 동결**은 유지해야 한다.
