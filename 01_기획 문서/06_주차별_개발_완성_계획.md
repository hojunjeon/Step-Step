## 2. 진척도 정의

진척도는 작성한 코드 줄 수가 아니라 **통합 증거**로 평가한다.

| 진척도 | 상태 정의 | 요구 증거 |
|---:|---|---|
| 0% | 아이디어 | 문서·논의만 존재 |
| 10% | 인터페이스 정의 | 스키마·핀맵·토픽·담당자 확정 |
| 20% | 단품 동작 | 센서 스트림, 모터 단독, API 단독 |
| 40% | 하위 기능 완성 | 한 모듈이 실제 입력으로 동작 |
| 60% | 최초 종단 간 | 로봇 이벤트가 서버 지도까지 1회 연결 |
| 75% | 평가 시나리오 연결 | 루트·AI·회피·서버가 한 코스에서 동작 |
| 90% | 반복 안정 | 3회 연속 성공, 치명 오류 없음 |
| 100% | 동결·재현 가능 | 설치 절차·백업·시연·문서까지 완료 |

### 상태 색상

- **Green:** 주차 게이트를 기한 내 통과
- **Yellow:** 1~2일 지연, 검증된 폴백 존재
- **Red:** 핵심 게이트 2일 초과 지연 또는 폴백 없음

Red가 되면 기능을 추가하지 않고 문서의 `범위 축소 순서`를 즉시 적용한다.
