## 1. 배경과 문제 정의

시각장애인은 선형블록의 방향과 점형블록의 경고 정보를 발바닥 또는 흰지팡이로 확인한다. 점자블록이 파손되거나 돌기가 마모되고, 단차가 생기거나 경로가 장애물에 가려지면 안내 기능이 저하된다.

지자체가 보수계획을 세우려면 현장 조사자가 넓은 구역을 반복 순회하며 다음 정보를 직접 수집해야 한다.

- 불량 위치
- 불량 유형과 규모
- 사진
- 주변 보행환경
- 반복 발생 여부
- 예상 보수 구간과 물량
- 과거 보수 이력

현장 조사 결과가 단발성 문서나 민원 단위로 남으면 구간별 악화 추세와 예산 우선순위를 비교하기 어렵다. 본 제품은 이 조사 단계를 로봇 순찰로 보조하고, 원본 이미지를 **관리 구간과 보수 후보 공사 구간**으로 가공하는 것을 목표로 한다.

## 2. 제품 비전과 목표

### 2.1 제품 비전

> 사람이 모든 구간을 직접 반복 조사하지 않아도 점자블록의 상태 변화와 보수 우선순위를 근거 데이터로 확인할 수 있게 한다.

### 2.2 정량 목표

평가용 MVP에서는 다음을 목표로 한다.

| 지표 | 평가 목표 |
|---|---:|
| 지정 루트 완주율 | 통제 코스에서 3회 연속 완주 |
| 점자블록 정상 추종 | 직선·완만한 곡선 구간 90% 이상 유지 |
| 장애물 정지 | 전방 안전영역 침범 장애물 100% 정지 |
| 장애물 회피 후 복귀 | 지정 시나리오 5회 중 4회 이상 |
| 대표 이미지 중복 촬영 | 동일 결함 통과 1회당 최대 1개 이벤트를 원칙으로 하며, 10회 중 9회 이상 |
| 이벤트 전달 | 온라인 시 30초 이내 지도 반영, 오프라인 후 재연결 시 누락 0건 |
| 지도 분류 | 4종 불량 + 장애물의 5종 아이콘 표시 |
| 데이터 집계 | 인접 이벤트를 3~5 m 관리 구간으로 자동 묶음 |
| 안전 | 통신 끊김·프로세스 정지·비상정지 입력 시 구동 출력 차단 |

실제 현장 배치 전에는 보행자 혼재 환경에 대한 별도 안전 검증, 보험·도로 점용·개인정보 처리 검토가 필요하다. 평가 MVP는 통제된 코스에서 검증한다.

## 4. 서비스 원칙

### 4.1 즉시 민원보다 누적 조사

장애물 또는 불량 한 건을 발견했다고 즉시 행정 조치를 요청하지 않는다. 반복 순찰 데이터를 이용해 다음을 구분한다.

- 일시적 장애물
- 반복 장애물
- 진행이 멈춘 경미한 손상
- 빠르게 악화되는 손상
- 여러 불량이 밀집한 공사 우선 구간

### 4.2 개별 점보다 구간 중심

GPS가 순찰마다 수 m 달라질 수 있으므로 개별 블록의 영구 식별을 필수로 하지 않는다. 지도상의 근접성과 사전 루트 상 누적 거리를 이용해 3~5 m 관리 구간으로 묶는다.

### 4.3 AI 판단은 근거와 신뢰도를 함께 제공

지도에는 단일 위험도 숫자만 표시하지 않는다.

- 위험도
- 분석 근거
- 모델 신뢰도
- 반복 관측 횟수
- 마지막 관측일
- 현장 확인 필요 여부

추천 교체 시기는 장기 데이터가 부족할 때 `즉시`, `3개월 이내`, `6개월 이내`, `다음 정기점검`, `추세 관찰`과 같은 구간값으로 제공한다.

### 4.4 로봇 안전이 탐지보다 우선

불량 촬영을 위해 위험한 움직임을 하지 않는다. 장애물 또는 센서 이상이 발생하면 촬영 성공 여부와 관계없이 정지가 우선이다.

## 6. 기능 범위

## 6.1 온디바이스 기능

### FR-E01. 루트 관리

- 루트 생성, 이름 지정, 버전 관리
- 루트 1, 루트 2 등 목록 표시
- 순찰 전 루트 선택
- 루트별 시작점·종료점·예상 거리·주의 구간 저장
- 수동 조이스틱 주행으로 최초 루트 기록
- 선형 점자블록 구간, 갈림길, 점자블록 단절 구간, 촬영 주의 구간을 세그먼트 속성으로 저장

### FR-E02. 자율주행

- 정상 시 점자블록 중심선 추종
- 갈림길에서 사전 루트의 다음 진행 방향 선택
- 점자블록이 끊기면 사전 경로와 로컬 위치추정으로 연결 구간 통과
- LiDAR 기반 충돌 정지
- 장애물 정지 후 사진 촬영
- 저속 회피 후 가장 가까운 사전 경로로 복귀
- 경로 복귀 실패 시 정지하고 운영자 개입 요청

### FR-E03. 불량 탐지

평가 MVP의 온디바이스 클래스는 다음 네 가지다.

| 내부 코드 | 화면 표시 | 의미 |
|---|---|---|
| `SURFACE_WEAR` | 마모·벗겨짐 | 표면 박리, 돌기 손실, 마모 후보 |
| `CRACK` | 크랙 | 선형 균열 및 균열 집합 |
| `BREAKAGE` | 파손 | 깨짐, 결손, 구조적 손상 |
| `STEP_CANDIDATE` | 단차 | 카메라상 단차 후보, 하향 거리센서로 보조 확인 |

- YOLOv8n-seg로 결함 영역을 마스크 단위 탐지
- 객체가 중앙 촬영 영역에 들어왔을 때 대표 프레임 선정
- 연속 프레임 다수결과 추적으로 최종 클래스 확정
- 긴 크랙은 방향·근접도·연결성을 기준으로 하나의 이벤트로 병합
- 촬영 품질이 낮으면 한 차례 재촬영
- 이미지, 클래스, 신뢰도, GPS, 루트 ID, 루트 상 누적거리, 촬영 방향 전송

### FR-E04. 장애물 이벤트

- 장애물 종류는 분류하지 않는다.
- LiDAR 전방 안전영역에 객체가 3회 연속 검출되면 정지
- 정지 후 카메라로 대표 이미지 촬영
- `[이미지, OBSTACLE, GPS 좌표, 루트 ID, 누적거리, 시간]` 저장·전송
- 주행 가능 폭이 확보되면 회피
- 회피 후 사전 경로에 재투영하여 정상 주행
- 회피 공간이 부족하거나 LiDAR 이상이면 정지 유지

### FR-E05. 오프라인 동작

- 네트워크 장애 시 이미지와 메타데이터를 로컬 `outbox`에 저장
- 이벤트 UUID와 이미지 해시로 중복 전송 방지
- 연결 복구 시 오래된 이벤트부터 재전송
- 업로드 완료 전 원본을 삭제하지 않음
- 디스크 사용량이 임계치를 넘으면 저품질 중간 프레임부터 정리하고 대표 이벤트는 보존

### FR-E06. 기기 상태

- Jetson·Raspberry Pi 연결 상태
- 카메라·LiDAR·GNSS·IMU·엔코더 상태
- 배터리 전압
- 현재 상태 머신
- 마지막 서버 연결 시각
- 업로드 대기 이벤트 수
- 비상정지 상태

## 6.2 서버 기능

### FR-S01. 이벤트 수집

- 이미지 업로드용 사전 서명 URL 발급
- 메타데이터 수신
- 이벤트 UUID 기반 멱등 처리
- 원본 분류와 모델 버전 저장
- 장애물과 불량 이벤트 분리 처리

### FR-S02. VLM 상세 분석

Qwen2.5-VL-7B-Instruct 기반 서비스가 다음 구조화 결과를 생성한다.

- 결함 상세 설명
- 결함 면적 비율
- 돌기 기능 저하 추정
- 크랙 폭·길이 구간
- 단차 보조센서 값
- 색상 대비·변색 속성
- 위험도 0~100
- 추천 조치
- 추천 교체 시기 범주
- 분석 신뢰도
- 현장 검토 필요 여부

평가 단계에서는 완전한 파인튜닝보다 고정 JSON 스키마, 예시 기반 프롬프트, 규칙 결합을 우선한다.

### FR-S03. 지도 시각화

지도 아이콘은 총 5종이다.

| 아이콘 | 조건 |
|---|---|
| 마모·벗겨짐 | `SURFACE_WEAR` |
| 크랙 | `CRACK` |
| 파손 | `BREAKAGE` |
| 단차 | `STEP_CANDIDATE`가 센서 또는 VLM 확인을 통과 |
| 장애물 | `OBSTACLE` |

상세 패널에는 다음을 표시한다.

- 대표 이미지
- 최근 관측 시각
- GPS 좌표
- 루트명과 루트 상 거리
- YOLO 클래스·신뢰도
- VLM 분석
- 위험도·추천 교체 시기
- 반복 관측 횟수
- 검토 상태
- 원본 이벤트 이력

### FR-S04. 관리 구간 집계

- 같은 루트에서 3~5 m 범위의 이벤트를 하나의 관리 구간으로 집계
- 클래스별 개수와 최대·평균 위험도 계산
- 반복 순찰 간 상태 변화 계산
- 주변 결함 밀도에 따라 보수 후보 구간 확장
- 공사 단위 길이, 손상 블록 수, 주요 결함, 우선순위 제공

### FR-S05. 보수 계획 지원

```text
보수 우선순위 =
결함 위험도
× 악화 속도 보정
× 반복 검출 신뢰도
× 구간 중요도
× 보행 영향 범위
```

- 지하철역·버스정류장·병원·복지시설 인접성 등 구간 중요도 입력
- 부분 보수/구간 교체 후보
- 예상 블록 수와 구간 길이
- 우선 검토 목록
- 보수 전·후 이미지 비교
- 보수 완료 후 다음 순찰에서 자동 재검사

## 6.3 관리자 웹 기능

### 화면 1. 대시보드

- 오늘/이번 주 순찰 수
- 루트 완주율
- 신규 불량·장애물 수
- 업로드 지연
- 높은 위험도 구간
- 악화 속도가 빠른 구간

### 화면 2. 지도

- 5종 아이콘 필터
- 날짜·루트·위험도·검토 상태 필터
- 결함 현황, 보수 우선순위, 설치 적정성, 악화 추세, 보수 이력 레이어
- 아이콘 클러스터링
- 관리 구간 폴리라인 표시

### 화면 3. 이벤트 상세

- 원본 이미지와 마스크
- YOLO 결과와 VLM 결과 비교
- 과거 관측 타임라인
- 담당자 검토·오탐 처리
- 보수 대상 편입

### 화면 4. 보수 계획

- 보수 후보 구간 목록
- 예상 길이·블록 수
- 우선순위

## 7. 데이터 모델의 핵심 원칙

### 7.1 원본 이벤트 보존

AI 분석값이 변경되어도 원본 이미지, 원본 YOLO 결과, 센서값, 모델 버전은 보존한다.

### 7.2 이벤트와 시설 상태 분리

- **Event:** 특정 시각의 한 번의 관측
- **Maintenance Segment:** 여러 관측을 묶은 3~5 m 관리 구간
- **Repair Package:** 실제 공사 계획에 사용할 연속 구간

### 7.3 위치 정확도의 현실적 처리

GPS는 현장 접근과 지도 표시에 사용한다. 점자블록 폭 수준의 정밀도를 요구하지 않는다. 동일 루트의 누적거리와 지도 근접성을 함께 사용해 구간을 묶는다.

### 7.4 분류와 보수 판단 분리

`CRACK`은 관측 클래스이며 `즉시 교체`는 행정·안전 판단이다. YOLO가 클래스를 찾고, VLM·규칙·시계열 알고리즘이 위험도와 추천 조치를 산출한다.

## 8. MVP와 후속 범위

| 기능 | 평가 MVP | 후속 고도화 |
|---|---|---|
| 루트 | 1~2개 고정 루트 기록·선택 | 여러 구역·자동 배차 |
| 주행 | 저속 통제 코스, 점자 추종+사전 경로 | 실제 보도, 동적 보행자 환경 |

| 변색 | VLM 속성 | 독립 클래스·색 대비 정량화 |
| 단차 | 카메라 후보+ToF 보조 | 깊이 카메라/3D 측정 |
| 장애물 | 존재 여부, 정지·회피 | 높이·지속성·보행 폭 분석 |
| 위험도 | 규칙+VLM 잠정값 | 전문가 라벨 기반 파인튜닝 |
| 교체 시기 | 범주형 추정, 신뢰도 표시 | 장기간 시계열 예측 |

| 개인정보 | 업로드 전 또는 서버에서 얼굴·번호판 블러 | 정책 기반 자동 보존기간 |

## 9. 비기능 요구사항

### NFR-01. 안전

- 통신 명령이 0.5초 이상 끊기면 구동 정지
- 물리 비상정지 버튼은 모터 전원 또는 드라이버 Enable을 직접 차단
- 시연 최고속도 0.35 m/s
- 회피 속도 0.20 m/s 이하
- 센서 상태 불량 시 자율주행 금지
- 수동 조이스틱 모드로 즉시 전환 가능

### NFR-02. 신뢰성

- 이벤트 UUID 멱등성
- 오프라인 저장
- 프로세스 자동 재시작
- 순찰 로그와 상태 전이 기록
- 설정·모델·루트 버전 고정

### NFR-03. 성능

- 점자블록 추종 추론 10 FPS 이상
- 조향 명령 20 Hz 이상
- LiDAR 충돌 판단 5 Hz 이상
- 온라인 이벤트 지도 반영 30초 이내
- 웹 지도 일반 조회 3초 이내

### NFR-04. 설명 가능성

- 위험도 산출 요소를 함께 표시
- 모델 버전과 신뢰도 저장
- 담당자가 오탐·수정 가능
- 추천 교체 시기를 확정 행정 명령으로 표현하지 않음

### NFR-05. 개인정보

- 얼굴·차량 번호판은 저장 전 블러 또는 서버 비동기 블러
- 원본 접근 권한 분리
- 보존기간 설정
- 시연 데이터는 통제된 공간에서 촬영

## 10. 평가 시연 시나리오

### 코스

- 직선 점자블록
- 완만한 곡선
- 점자블록 단절 구간
- 갈림길
- 마모·크랙·파손·단차 모형
- 점자블록 위 고정 장애물
- 장애물 우회 후 복귀 지점

### 시연 흐름

1. 7인치 화면 또는 웹에서 `ROUTE_DEMO_01` 선택
2. 센서 사전 점검 결과 표시
3. 로봇 출발
4. 점자블록 추종
5. 크랙 모형 중앙 진입 시 감속·촬영
6. 장애물 접근 시 정지
7. 장애물 이벤트 생성 후 회피
8. 점자블록 단절 구간에서 사전 경로 추종
9. 종료점 도착
10. 웹 지도에서 이벤트 5종, 이미지, 위험도 조회
11. 동일 위치의 과거 데이터와 구간 추세 표시
12. 네트워크를 잠시 차단한 이벤트가 재연결 후 업로드되는 장면 제시

### 발표 핵심 문장

> 이 로봇은 개별 민원을 자동 신고하는 장비가 아니라, 반복 순찰 데이터를 구간 단위 보수 후보로 변환하여 지자체 조사와 예산 수립을 보조하는 시스템이다.

## 12. 제외 범위

평가 MVP에서는 다음을 목표로 하지 않는다.

- 공공 보도에서 무감독 장시간 운행
- 사람·차량·자전거 등 장애물 종류 분류
- 법적 효력을 갖는 시설 적합 판정
- 장기간 데이터 없이 정확한 교체일을 단일 날짜로 예측
- RTK-GNSS 수준 위치 정확도
- 완전한 다중 로봇 관제
- Kubernetes 자체를 평가 핵심 기능으로 시연

이 범위를 명확히 해야 네 주 내 통합 가능한 제품으로 완성할 수 있다.
