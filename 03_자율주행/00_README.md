# 자율주행 고도화 가이드

## 목적

이 폴더는 현재의 규칙 기반 자율주행을 안전하게 검증한 뒤, 자체 주행 데이터와 사전학습 모델을 이용해 AI 보조 주행으로 단계적으로 고도화하는 방법을 설명한다.

핵심 원칙은 **기본 주행 완성 → 데이터 확보 → 작은 모델 검증 → 안전한 보조 적용 → 엣지 배포 최적화** 순서를 지키는 것이다. AI 모델은 충분한 폐루프 검증 전까지 독립 Safety Supervisor를 우회하거나 기존 Planner를 직접 대체하지 않는다.

## 기초 설명 문서의 공통 목차

1. **한 문장으로 설명하면** — 전문용어 없는 정의
2. **왜 필요한가** — 해결하려는 실제 문제
3. **일상적인 비유** — 사람의 눈·귀·두뇌·운전 행동과 연결
4. **어떻게 동작하는가** — 입력부터 출력까지 3~5단계
5. **우리 로봇에서는** — 점자블록 순찰 상황의 구체적인 예
6. **그림으로 보기** — 간단한 구성도 또는 데이터 흐름
7. **자주 헷갈리는 점** — 비슷한 개념과 차이
8. **핵심 용어** — 처음 나온 용어를 쉬운 말로 다시 정의
9. **한 페이지 요약** — 반드시 기억할 내용 3~5개
10. **확인 질문** — 다음 문서로 넘어가기 위한 짧은 자가 점검

## 구현 가이드 문서의 공통 목차

1. **이 단계가 필요한 이유** — 앞 단계와 연결되는 쉬운 설명
2. **단계 목표** — 해결할 문제와 아직 해결하지 않을 문제
3. **선행 조건** — 필요한 하드웨어, 데이터, 이전 단계의 통과 기준
4. **입력과 출력** — 센서·데이터·모델 입력 및 생성 산출물
5. **구현 가이드** — 작업 순서, 설정, 명령, 핵심 인터페이스
6. **검증 방법** — offline·replay·shadow·closed-loop 중 해당 시험 절차
7. **완료 기준** — 수치로 판정 가능한 체크리스트와 Go/No-Go 기준
8. **안전 및 실패 대응** — 중지 조건, fallback, 수동 개입, 로그 보존
9. **다음 단계 인계** — 다음 문서가 사용할 산출물과 미해결 항목

## 권장 진행 방식

### 1. 기초 문서를 먼저 완성한다

`01`부터 `07`까지는 특정 라이브러리 사용법보다 개념과 프로젝트 전체 흐름을 설명한다. 독자가 `인지 → 위치 추정 → 판단·계획 → 제어 → 안전 감시`의 관계를 자기 말로 설명할 수 있으면 구현 구간으로 넘어간다.

### 2. 첫 구현 단계의 범위를 확정한다

`10_규칙기반_차량완성_및_데이터수집.md`에서 센서 topic, timestamp 동기화, TF, 조향·속도 보정, E-stop, watchdog, 기록할 episode metadata를 구체화한다. 이 기반이 정해지기 전에는 모델 선정이나 학습 코드를 먼저 확장하지 않는다.

### 3. 한 단계씩 문서와 증거를 함께 완성한다

문서 초안을 쓴 뒤 실제 설정, 로그 예시, 시험 결과를 채우고 완료 기준을 통과시킨다. 통과하지 못한 항목은 다음 단계의 전제로 넘기지 않고 현재 단계의 미해결 항목으로 남긴다.

### 4. 모델 비교와 실차 제어를 분리한다

사전학습 모델은 `replay → shadow → 제한적 assist` 순서로 검증한다. 공개 checkpoint의 waypoint를 곧바로 모터 명령으로 연결하지 않는다.

### 5. 데이터 규모에 맞춰 권한을 높인다

- 데이터가 없을 때: 로거와 공개 checkpoint 실행만 검증
- 최소 자체 episode 확보 후: frozen encoder와 작은 head를 offline·shadow 평가
- 다양한 장소·날짜·실패 사례 확보 후: Safety Supervisor 뒤에서 제한적 assist 평가
- 충분한 폐루프 근거 확보 후: 통제 코스에서 Planner 일부 대체를 검토

### 6. 최적화는 정확한 모델이 나온 뒤 수행한다

Jetson 배포는 `FP32 기준선 → FP16 TensorRT → INT8 PTQ → 필요 시 QAT 또는 증류` 순서로 진행한다. 각 변환 단계에서 정확도, 지연, GPU·RAM 사용량, timeout을 함께 기록한다.

## 문서 간 공통 안전 기준

- 학습 모델과 독립된 충돌 정지, E-stop, watchdog을 유지한다.
- 센서 시간 불일치, TF 단절, 추론 timeout, 위치 점프 시 감속 또는 정지한다.
- AI 출력은 Safety Supervisor의 승인 후에만 차량 제어 계층으로 전달한다.
- 평균 오차 하나만으로 성공을 선언하지 않고 완주율, 개입률, 충돌·정지, 우회 후 복귀 성공률을 함께 본다.

## 현재 문서 상태와 다음 작업

기초 이해 `01~04`, 방식 이해 `05~07`, 구현 가이드 `10~15` 문서가 작성됐다. 이제 문서 수를 늘리기보다 `10_규칙기반_차량완성_및_데이터수집.md`의 체크리스트를 실제 장비 기준으로 채운다.

1. 실제 ROS 2 topic 이름·message type·주기를 확정한다.
2. TF와 카메라·LiDAR calibration version을 기록한다.
3. E-stop·watchdog·LiDAR 정지를 실제로 시험한다.
4. 2~5분짜리 짧은 ROS bag 10개를 기록하고 재생한다.
5. 데이터 품질 Gate를 통과한 뒤에만 사전학습 모델 비교로 넘어간다.
